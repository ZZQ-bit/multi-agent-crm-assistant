# CordysCRM 风险信号参考

## 用途

本文档整理 CordysCRM skill 中的风险识别规则，作为 多Agent智能助手 多 Agent 分析的知识库材料。它不代表最终审批制度，只用于帮助 Agent 发现可疑信号、解释风险来源，并给出需要人工复核的下一步建议。

AI 的职责是生成判断和草稿，不替销售、财务、交付、商务或法务做最终审批。

## 风险识别原则

- 不把所有异常都堆给用户，一次优先呈现最关键的 1-3 条。
- 判断比复述重要，要说明异常对成交、回款、交付或合同责任的影响。
- 区分“确实没有数据”和“数据获取失败”。接口失败时不能生成确定性风险结论。
- 风险信号只用于建议人工复核，不直接阻断业务流程。
- 输出中不要暴露原始接口响应、内部 Prompt、密钥或调试信息。

## 销售推进风险

| 信号 | 风险含义 | 建议动作 |
| --- | --- | --- |
| 线索超过 3 天未跟进 | 线索热度可能下降 | 提醒销售今日优先联系 |
| 商机在当前阶段停留超过 7 天 | 商机推进卡顿 | 确认卡点、决策人和下一步动作 |
| 今日跟进计划未完成 | 执行动作延误 | 整理待办优先级 |
| 线索积压超过 20 条 | 个人线索池可能失控 | 建议筛选高价值线索优先处理 |
| 客户超过 90 天无跟进 | 客户活跃度下降 | 建议安排复联或重新评估客户价值 |
| 客户有活跃商机但超过 60 天无跟进 | 商机可能失控 | 建议补充跟进记录并确认推进计划 |

## 财务与回款风险

| 信号 | 风险含义 | 建议动作 |
| --- | --- | --- |
| 回款逾期 | 现金回收风险增加 | 由销售确认客户付款安排，财务跟进催收 |
| 未来 7 天有多笔回款到期 | 短期回款压力集中 | 提前确认付款节点 |
| 合同已签约但无回款计划 | 合同到现金链路断裂 | 补建回款计划或确认合同条款 |
| 合同签约超过 30 天无回款记录 | 履约后回款风险 | 核查客户付款状态 |
| 回款计划已到期但无回款记录 | 计划执行偏差 | 标记逾期并推动催收 |
| 已开发票但未回款 | 开票后应收风险 | 核查付款进度和账期 |
| 已回款但未开发票 | 发票流程缺口 | 提醒商务或财务补充开票流程 |

## 合同与商务风险

| 信号 | 风险含义 | 建议动作 |
| --- | --- | --- |
| 合同审批超过 7 天未完成 | 合同流程卡顿 | 追踪当前审批节点 |
| 合同被同一节点驳回多次 | 条款或资料存在反复问题 | 汇总驳回原因后再重新提交 |
| 合同到期小于等于 30 天且未续约 | 续约风险 | 提醒销售确认续约意向 |
| 签约合同缺附件 | 合同归档不完整 | 补齐附件或审批记录 |
| 合同存在特殊条款 | 法务责任边界不清 | 法务或商务复核验收、赔付、责任条款 |

## L2C 链断裂风险

| 断裂点 | 条件 | 严重度建议 | 说明 |
| --- | --- | --- | --- |
| 线索到客户 | 线索创建超过 30 天未转化 | 警告 | 长期未转化可能占用线索池 |
| 商机到合同 | 商机赢单但未找到关联合同 | 警告/严重 | 赢单后未签约可能存在丢单或漏建风险 |
| 合同到回款计划 | 合同已签约但无回款计划 | 严重 | 影响现金回收管理 |
| 回款计划到回款记录 | 计划到期但无回款记录 | 警告/严重 | 表示付款未按计划执行 |
| 合同到发票 | 合同签约后长期未开发票 | 警告 | 发票流程可能滞后 |
| 发票到回款 | 已开发票但未回款 | 警告/严重 | 形成应收账款压力 |

## 在各 Agent 中的使用

### 销售 Agent

关注商机阶段停留、客户活跃度、最近跟进、下一步计划和销售推进卡点。输出应具体到“联系谁、确认什么、下一步怎么推进”。

### 财务 Agent

关注付款条件、账期、回款计划、实际回款、开票状态和应收压力。输出应明确金额、日期、节点和需要财务复核的原因。

### 合同 Agent

关注合同审批、条款、验收、赔付、到期、附件和审批卡点。输出应提示商务或法务复核，不给最终法律意见。

### 交付 Agent

CordysCRM skill 对交付规则覆盖较弱，交付 Agent 应优先使用本项目的 `02-交付与定制规则.md`，仅在合同、订单或商机链路中补充查看交付相关信号。

## 多Agent智能助手 输出方式

风险信号不要直接以“规则表”塞给用户，应转成自然业务结论：

```text
结论：当前方案回款风险偏高。
原因：首付款比例偏低，同时尾款安排在验收后支付；如果合同签约后未及时建立回款计划，后续催收难度会增加。
建议：先确认首付款比例和回款节点，再由财务复核付款条件。
```

判断过程可展示为：

```text
判断过程
- 已读取商机付款条件
- 已参考合同到现金链路规则
- 已发现尾款验收后支付
- 已生成财务复核建议
```
