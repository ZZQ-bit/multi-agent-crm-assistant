# 多Agent智能助手 文档索引

## 1. 当前文档基线

本文档是 `docs/` 目录的当前入口。后续 Agent 或协作者进入项目时，应先阅读本文档，再阅读 `01-多Agent智能助手 PRD.md`。

当前阶段的唯一产品主线是 `多Agent智能助手`，不是重做 `CordysCRM`。`CordysCRM` 是 CRM 业务底座，负责承载客户、商机、联系人、跟进记录和跟进计划等对象；`多Agent智能助手` 是嵌在 CRM 内的 AI 商机协同工作台。

当前已确认的产品口径：

- 页面结构以当前双栏工作台为准：左侧历史会话 + 中间对话主工作区，不做固定右侧业务栏。
- P0 必须具备真实 CRM 写回能力，但写回只在明确写回意图和用户确认后触发。
- AI 负责生成判断、建议和草稿，不直接替用户审批，不自动执行高风险业务动作。
- P0 以 Agent 能力契约和自主路由为核心，不以固定演示链路为核心。

如果旧文档与本文档或 `01-多Agent智能助手 PRD.md` 冲突，以本文档和最新版 PRD 为准。

## 6. 冲突处理规则

当文档之间出现冲突时，按以下优先级判断：

1. `docs/01-多Agent智能助手 PRD.md`
2. `docs/00-文档索引.md`
3. `docs/00-多Agent智能助手 重构启动说明.md`
4. 当前可用支撑文档
5. `docs/archive/` 和 `docs/reference/` 中的历史或参考资料

特别说明：

- 任何把写回 CRM 写成默认结尾、固定链路或 mock 写回的旧表述，都已被新 PRD 和 `10-实施路线与验收清单.md` 替代。
- 任何要求 P0 固定三栏或固定右侧业务栏的旧表述，都已被当前双栏工作台口径替代。
- 原型页面只展示真实用户会看到的业务内容，不展示技术标签、调试说明、内部状态名或设计备注。
- 页面文案使用销售用户语言，例如“选择商机”“判断过程”“确认写入”，避免 `mock`、`quick_answer`、`full_review`、`context` 等内部表达。

## 7. 后续 Agent 协作规则

后续协作者或 Agent 应遵守：

- 先读 `00-文档索引.md` 和 `01-多Agent智能助手 PRD.md`，再读其他资料。
- 不默认引用 `archive/` 下旧文档作为当前方案依据。
- 如果实现、原型或 Chatflow 与 PRD 冲突，应先指出冲突，再提出可选修正路径。
- 遇到产品方向分歧、关键假设不成立、真实写回不可行或演示闭环受阻时，必须及时抛出问题。
- 文档重构优先服务 Agent 能力契约：主 Agent 自主路由，按需调用轻量回答、CRM 查询、专项分析、多 Agent 评审、内容生成和写回确认能力。
