# 多Agent智能助手 PRD

## 1. 产品定位

`多Agent智能助手` 是嵌入 CordysCRM 的销售业务助手，帮助销售在 CRM 内用自然语言完成客户/商机查询、进展总结、风险判断、沟通内容生成和跟进计划草稿生成。

`CordysCRM` 是业务底座，不是本项目要重点重做的产品。它提供客户、商机、联系人、跟进记录、跟进计划等业务对象和基础业务操作能力。`多Agent智能助手` 基于这些对象判断任务意图，并按需组合 CRM 查询、业务规则、多 Agent 分析和内容生成能力。

产品不定位为：

- 通用聊天机器人。
- 单一知识库问答入口。
- 自动审批工具。
- 自动销售决策系统。
- 固定演示脚本或固定流程流水线。
- 独立于 CRM 的外部 AI 应用。

## 2. 目标用户

| 用户 | 核心诉求 | P0 关注点 |
| --- | --- | --- |
| 一线销售 | 快速理解客户和商机，形成下一步动作 | 查询上下文、总结进展、生成建议和草稿 |
| 销售主管 | 判断重点商机质量和成交风险 | 查看结论、风险和下一步建议 |
| 财务/商务 | 判断折扣、付款、回款风险 | 识别付款和折扣风险 |
| 交付/实施 | 判断上线周期、范围和资源风险 | 识别交付承诺风险 |
| 合同/法务支持 | 判断条款、验收和责任边界风险 | 识别合同风险 |

P0 主要面向一线销售和销售主管。财务、交付、合同视角通过专项 Agent 或多 Agent 分析体现，不要求这些角色分别登录独立工作台。

## 3. 核心业务能力

多Agent智能助手 P0 支持多类自由问法。Agent 不应把所有问题强行套入固定流程，而应先判断用户意图，再选择合适能力组合。

| 能力 | 说明 | P0 要求 |
| --- | --- | --- |
| 轻量回答 | 回答非 CRM 或泛销售管理问题 | 不声称已查询 CRM |
| CRM 对象查询 | 查询客户、商机、联系人、跟进记录、跟进计划 | 对象不唯一时要求用户选择 |
| 商机/客户总结 | 汇总对象进展、风险、下一步 | 不默认生成写回草稿 |
| 专项风险判断 | 分析财务、交付、合同或销售推进风险 | 按问题只调用必要视角 |
| 多 Agent 评审 | 对复杂商机进行多视角综合判断 | 高复杂度问题才触发 |
| 内容生成 | 生成沟通话术、跟进摘要、计划草稿等 | 不声称已写入 CRM |
| CRM 写回草稿 | 用户明确要求保存、创建或写入时进入 | 生成可复制草稿，不声称已写回 |

复杂商机评审是当前版本的高复杂度能力，但不是所有商机问题的默认路径。当前 Chatflow 不执行 CRM 写入。

## 4. P0 Agent 能力模型

P0 的核心不是一条固定闭环，而是一套可按需组合的 Agent 能力模型：

```text
用户自由提问
  -> 主 Agent 判断任务意图、业务对象、信息缺口和风险复杂度
  -> 按需选择轻量回答、CRM 查询、销售协同、专项风险、多 Agent 评审或内容生成
  -> Agent 根据任务输出合适形态的业务回答
  -> 出现显式写回意图时生成可复制草稿，并说明当前未执行写入
```

Agent 可自主决定：

- 是否需要查询 CRM。
- 是否需要让用户选择客户或商机。
- 是否需要追问补充条件。
- 是否只需轻量回答、单 Agent 专项分析，还是多 Agent 评审。
- 输出应该是短答、摘要、建议清单、风险判断还是内容草稿。

Agent 必须遵守：

- 不编造 CRM 数据。
- 不把内部路由、Prompt、工具调用、技术标签展示给业务用户。
- 不把写回草稿作为默认输出。
- 不声称执行当前 Chatflow 未完成的 CRM 写入。
- 对象不唯一时不猜测。
- 信息不足时说明不确定性，必要时追问。

## 5. 页面结构与交互原则

P0 页面采用当前双栏工作台结构：

| 区域 | 职责 |
| --- | --- |
| 左侧历史栏 | 新建会话、查看历史会话、切换当前会话 |
| 中间对话主工作区 | 用户提问、AI 回答、判断过程、多结果文本澄清、附件和输入区 |

P0 不做固定右侧业务栏。客户、商机、联系人、跟进记录和跟进计划只在以下位置出现：

- 用户通过输入框绑定或选择业务对象。
- 对象不唯一时，在对话流中展示 Markdown 客户/商机列表，并提示用户输入完整名称继续。
- AI 回答中引用当前业务对象和关键依据。
- 用户点击查看详情时跳转到 CRM 已有详情页。

交互原则：

- 对话是入口，Agent 能力是核心。
- 普通问答保持轻量展示。
- 业务任务可展示“判断过程”，但不展示模型原始思维链。
- 输出结构由 Agent 根据任务决定，不要求所有回答套同一模板。
- 用户要求写回时，只生成目标对象、类型和内容明确的草稿。
- 当前页面不展示已经执行写回的成功状态。
- 页面不展示技术标签、内部状态名、调试说明或设计备注。

## 6. 多 Agent 分工

| Agent | 职责 | 输出边界 |
| --- | --- | --- |
| Planner | 理解用户问题，识别任务类型，确认业务对象，选择证据和 Agent | 不直接替用户审批或写回 |
| 轻量回答 LLM | 处理非 CRM 或泛销售管理问题 | 不声称查询了 CRM |
| 销售协同 Agent | 分析客户意向、商机阶段、推进动作和赢单机会 | 给推进判断和行动建议 |
| 财务判断 Agent | 分析金额、折扣、付款方式、回款和现金流风险 | 不输出财务已通过 |
| 交付判断 Agent | 分析上线周期、定制范围、资源和履约风险 | 不承诺具体资源或上线日期 |
| 合同判断 Agent | 分析合同条款、验收、赔付、责任边界和合规风险 | 不给最终法律意见 |
| 经营分析 Agent | 分析漏斗、转化、收入和回款数据 | 不编造统计指标 |
| CRM 业务回答生成 | 汇总证据和领域结论并生成 Markdown | 不逐份复述领域输出，不声称已写回 |

多 Agent 输出必须面向业务用户，不暴露 Prompt、节点名、内部路由标签或模型调度细节。

## 7. CRM 工具最小集

P0 需要具备以下 CRM 工具能力：

| 能力 | P0 要求 |
| --- | --- |
| 查询客户 | 根据名称、关键词或 ID 获取客户基础信息 |
| 查询商机 | 根据名称、关键词或 ID 获取商机金额、阶段、负责人和客户关系 |
| 查询联系人 | 获取商机关联联系人和关键角色 |
| 查询跟进记录 | 获取近期沟通记录和进展摘要 |
| 查询跟进计划 | 获取未完成计划和下一步安排 |
| 新建跟进记录 | 后端 Tool API 已提供；当前 YML 不调用 |
| 新建跟进计划 | 后端 Tool API 已提供；当前 YML 不调用 |

P0 不要求写回报价、审批、合同、正式财务结论或高风险商机字段。商机字段更新可在 P1 通过白名单和二次确认扩展。

## 8. 写回边界

当前 Chatflow 不调用 CRM 写入工具。普通查询、总结、风险分析和复杂评审不主动生成写回草稿；用户明确提出保存或写入要求时，系统生成可复制草稿并说明未执行写入。

AI 可以在合适任务中生成：

- 商机评审摘要。
- 风险判断。
- 下一步行动建议。
- 沟通话术或跟进文本。
- 跟进记录或跟进计划的待确认内容。

AI 不可以直接完成：

- 审批通过。
- 正式报价承诺。
- 合同可签结论。
- 财务已通过结论。
- 未经用户确认的 CRM 写回。

当前规则：

- 只有用户明确要求保存、写入、生成跟进记录或生成跟进计划时，才生成写回草稿。
- 草稿必须关联唯一客户或商机，并说明写入类型和内容。
- 后端写入接口和前端兼容字段不等于当前 Chatflow 已具备写回能力。
- 没有获得真实 CRM 记录 ID 时，不得返回写入成功。

## 9. P0 验收标准

P0 必须满足：

| 编号 | 验收项 | 标准 |
| --- | --- | --- |
| P0-01 | Agent 自主路由 | 能根据用户自由问法选择轻量回答、CRM 查询、专项分析、多 Agent 评审或内容生成 |
| P0-02 | 轻量回答可用 | 非 CRM 问题走轻量回答，不声称查询 CRM |
| P0-03 | 对象查询可用 | 用户能查询客户/商机；多候选时必须用 Markdown 文本澄清，不猜测对象 |
| P0-04 | 上下文查询可用 | AI 能基于 CRM 数据获取商机、客户、联系人、跟进记录和跟进计划 |
| P0-05 | 总结能力可用 | 能总结客户或商机进展，不默认生成写回草稿 |
| P0-06 | 专项判断可用 | 财务、交付、合同、销售推进类问题能按需走对应能力 |
| P0-07 | 多 Agent 评审可用 | 高复杂度商机问题可触发多视角分析和协调总结 |
| P0-08 | 内容生成可用 | 能生成沟通话术、跟进摘要或计划草稿，但不声称已写入 |
| P0-09 | 写回边界明确 | 显式写回意图下生成草稿，并明确当前未执行写入 |
| P0-10 | 失败可恢复 | 查询失败、对象不唯一或信息不足时提供可恢复处理 |

## 10. 非目标

P0 不做：

- 固定右侧业务栏。
- 固定演示脚本或固定输出模板。
- 把多 Agent 评审作为默认路径。
- 把写回草稿作为所有分析的默认输出。
- 完整商机评审详情页。
- 管理驾驶舱或多商机经营盘点。
- 报价、审批、合同等高风险真实写回。
- 自动审批或自动销售决策。
- 把 Dify、Prompt、工具调用、内部状态编码展示给业务用户。
