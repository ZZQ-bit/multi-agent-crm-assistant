# CordysCRM 角色视角参考

## 用途

本文档整理 CordysCRM skill 中的角色视角，用于指导 多Agent智能助手 的 Agent 分工、Prompt 风格和输出重点。多Agent智能助手 P0 面向销售工作台，但复杂商机评审需要模拟销售、财务、交付、合同等不同视角。

## 角色视角总览

| 角色 | 关注问题 | 典型输出 |
| --- | --- | --- |
| 销售 | 我接下来该做什么 | 行动优先级、跟进建议、沟通重点 |
| 销售经理 | 谁需要我关注 | 团队风险、卡点、需要介入的人和商机 |
| 高管 | 公司能交多少 | 趋势、目标差距、预测和重大风险 |
| 商务/合同 | 合同签对了没有 | 合同状态、审批卡点、条款和到期风险 |
| 财务 | 钱在哪 | 应收、回款、发票、逾期和现金链路 |

多Agent智能助手 P0 不需要完整支持经理和高管视角，但可以从中借用“风险升级”和“复核角色”的表达。

## 销售视角

### 核心关注

- 我的客户、线索和商机。
- 当前商机是否卡住。
- 客户最近有没有跟进。
- 今天和本周应该优先做什么。
- 下一步沟通对象、沟通内容和推进动作。
- 商机从线索到合同的转化是否顺畅。

### 输出风格

- 行动导向，少讲抽象判断。
- 具体到“联系谁、确认什么、什么时候做”。
- 先给下一步，再解释原因。
- 不替财务、合同或交付做最终结论。

### 用于销售 Agent 的提示

销售 Agent 应回答：

- 当前商机推进状态是什么。
- 主要卡点在哪里。
- 销售下一步应确认哪些信息。
- 是否需要拉财务、交付、商务或法务复核。
- 是否适合生成跟进记录或跟进计划草稿，但只有用户明确要求时才进入写回确认。

## 财务视角

### 核心关注

- 合同金额、付款条件、账期和回款节点。
- 回款计划是否完整。
- 是否存在逾期回款。
- 开票与回款是否匹配。
- 应收风险是否扩大。

### 输出风格

- 金额、日期和节点优先。
- 对风险等级保持谨慎，不给“可审批”结论。
- 说明为什么需要财务复核。
- 给出可谈判的付款调整建议。

### 用于财务 Agent 的提示

财务 Agent 应回答：

- 当前付款方案是否增加回款风险。
- 首付款、尾款、账期和验收后付款是否存在风险。
- 是否需要提高首付款比例或缩短账期。
- 是否需要补充回款计划、发票或合同数据。
- 哪些结论需要财务人工确认。

## 商务/合同视角

### 核心关注

- 合同审批状态。
- 合同条款是否完整。
- 验收、赔付、责任边界是否清晰。
- 合同到期、续约和附件归档。
- 签约后是否衔接开票和回款流程。

### 输出风格

- 强调流程完整性和条款风险。
- 不输出最终法律意见。
- 对验收、赔付、延期责任和数据安全等条款提示复核。
- 建议明确写入合同或补充协议的事项。

### 用于合同 Agent 的提示

合同 Agent 应回答：

- 哪些条款可能带来合同或履约风险。
- 验收口径是否清楚。
- 延期赔付和责任边界是否需要法务复核。
- 是否存在合同审批或附件完整性问题。
- 哪些内容应交由商务或法务确认。

## 交付视角

CordysCRM skill 对交付角色没有独立 profile。多Agent智能助手 的交付 Agent 应使用项目已有交付规则，并结合 CordysCRM 对象链路中的商机、合同、订单和跟进记录补充判断。

### 核心关注

- 上线周期是否合理。
- 定制范围是否明确。
- 验收口径是否可执行。
- 是否存在过度承诺。
- 是否需要分阶段交付。

### 输出风格

- 不承诺资源和上线日期。
- 给出范围冻结、阶段拆分和验收前置建议。
- 对不确定信息明确提示补充。

## 多 Agent 汇总视角

协调总结 Agent 不应该生成完整报告页，而应在 多Agent智能助手 P0 中输出轻量结论：

- 总体判断：风险低/中/高或信息不足。
- 关键风险：最多 3 条。
- 下一步建议：优先确认付款、交付、合同或销售推进事项。
- 复核角色：销售主管、财务、交付、商务/法务。
- 写回建议：仅当用户明确要求时，整理为跟进记录或跟进计划。

## 页面输出边界

- 不展示内部角色名作为技术标签。
- 可以说“从财务角度看”“从合同角度看”，但不要展示 `finance_agent`、`legal_agent` 等内部标识。
- 不展示 Prompt、工具请求体、模型思考过程。
- 不主动生成写回内容，除非用户明确要求保存或写入。
