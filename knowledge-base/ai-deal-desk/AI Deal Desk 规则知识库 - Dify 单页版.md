# AI Deal Desk 规则知识库 - Dify 单页版

文档说明：本页面是 AI Deal Desk 规则知识库的 Dify 单页版，用于适配 Dify 免费套餐一次只能选择一个 Notion 页面入库的限制。本文档合并财务风控、交付可行性、合同风险、销售推进、回答质量与 CRM 写回业务规则。Dify 入库时建议只选择本页面，不需要再选择 5 个分文档页面。

使用建议：分段模式选择通用，分段符选择两个换行，最大长度 1024，重叠长度 100，索引方式选择高质量，Embedding 模型选择 BAAI/bge-m3，检索方式选择混合检索，Rerank 模型选择 Qwen/Qwen3-Reranker-4B，Top K 设置为 8，暂时关闭 Score 阈值。

内容结构：下方内容会按财务规则、交付规则、合同规则、销售规则、回答质量与写回边界依次组织。每条规则尽量保持一个完整段落，便于 Dify 分段和语义召回。

规则治理说明：对象唯一性、不得伪造证据、不得替代审批、未经人工确认不得对折扣、付款、交付日期、合同条款或 CRM 写回作出承诺，属于 AI Deal Desk 的硬边界。财务、交付、合同和销售章节中的金额、比例、天数及风险等级，当前用于作品演示和内部评审参考；未标注来源、版本、适用范围和责任人的数值规则，不得表述为公司正式政策。若后续接入正式制度，每条正式政策应补齐来源、版本、适用范围和责任人。

综合判断说明：知识规则中的风险等级只代表该规则命中时的局部证据信号，不自动等同于整体商机风险。整体风险由 LLM 结合影响、发生可能性和可缓释性进行语义判断；事项优先级结合业务价值、时间窗口和推进成熟度判断，不使用关键词命中或代码评分。同一轮回答中的风险与优先级标签应保持一致。

## 01 财务风控规则

以下内容来源于 01-折扣与付款规则，用于财务风控 Agent。

文档说明：本文档用于支撑财务风控 Agent 判断商机中的折扣、付款、账期、分期、回款和信用风险。规则输出应作为内部评审建议，不代表审批通过、正式报价或财务最终意见。适用任务包括折扣审批判断、报价条件评审、付款方案评审、账期与回款风险判断、分期付款与验收尾款判断、客户信用风险提示、续费、增购、扩容和补充采购条件判断。不适用任务包括生成正式对外报价单、替代财务审批、替代授信或信用调查、承诺客户账期折扣或付款例外。

关键输入字段：财务风控 Agent 应优先读取客户名称、商机编号、商机金额或合同金额、折扣、产品范围、付款条款、首付款比例、账期天数、分期节点、验收与付款绑定条件、客户信用等级、历史欠款和应收状态、新购或续费增购类型。若金额、折扣、付款节点、账期、首付或客户信用任一关键字段缺失，不得输出低风险或可接受结论。

风险等级参考：低、中、高、阻塞用于描述单项财务条件的局部风险信号，不是整体商机的自动评级。财务 Agent 应结合影响、发生可能性、可缓释性以及证据完整度给出本轮判断；不得仅凭单条规则或关键词直接定级。缺少关键字段或触及禁止承诺边界时，应说明判断限制并提示人工确认。

规则 FIN-001：常规折扣复核。适用场景：折扣审批、报价条件评审。触发条件：折扣低于 85 折。风险等级：中。处理建议：要求补充客户价值、竞品态势、成交概率和替代报价方案，建议销售主管复核。Agent 使用要求：不得输出折扣已批准。

规则 FIN-002：深度折扣升级。适用场景：深折扣报价、特殊折扣审批。触发条件：折扣低于 80 折。风险等级：高。处理建议：建议升级至销售主管、财务负责人或授权管理者。Agent 使用要求：必须提示审批升级，不得输出可直接接受。

规则 FIN-003：超深折扣红线。适用场景：底价、毛利底线、超深折扣判断。触发条件：折扣低于或等于 70 折，或低于产品底价或毛利底线。风险等级：高。处理建议：标记为特殊例外，必须输出毛利影响、客户战略价值和替代方案。Agent 使用要求：不得仅因客户重要而降低风险等级。

规则 FIN-004：大额深折扣升级。适用场景：大金额商机折扣审批。触发条件：金额大于等于 500000 且折扣低于 85 折。风险等级：高。处理建议：要求同步销售主管和财务复核，输出审批材料清单。Agent 使用要求：必须把金额和折扣作为组合风险判断。

规则 FIN-005：折扣换条件。适用场景：折扣谈判、付款条件交换。触发条件：折扣低于 80 折但未换取年付预付、缩短账期、扩大合同周期或减少交付范围。风险等级：高。处理建议：建议用付款、合同期、范围收敛等条件交换折扣。Agent 使用要求：不得只分析折扣，不分析换取条件。

规则 FIN-006：实施费折扣限制。适用场景：实施费、定制费、迁移费减免。触发条件：实施费、定制费、迁移费被免除或低于成本覆盖线。风险等级：高。处理建议：建议保留实施成本覆盖，或减少交付范围。Agent 使用要求：必须联动交付成本判断。

规则 FIN-007：战略客户折扣材料。适用场景：战略客户、标杆客户、长期合作折扣理由。触发条件：用户以标杆客户、战略客户或长期合作作为折扣理由。风险等级：中。处理建议：要求补充可验证业务价值、标杆价值、复购潜力和成交概率。Agent 使用要求：不得仅因战略客户给出通过判断。

规则 FIN-008：首付款不足。适用场景：付款方案评审。触发条件：首付款比例低于 50%。风险等级：高。处理建议：建议提高首付，或将交付范围绑定付款节点。Agent 使用要求：不得忽略首付不足造成的现金流和交付投入风险。

规则 FIN-009：高金额低首付。适用场景：大额合同付款风险。触发条件：金额大于等于 500000 且首付低于 50%，或金额大于等于 1000000 且首付低于 70%。风险等级：高。处理建议：建议升级财务复核，并核算实施投入覆盖情况。Agent 使用要求：必须输出金额与首付叠加风险。

规则 FIN-010：SaaS 年付预付基线。适用场景：SaaS 订阅付款方式判断。触发条件：SaaS 订阅服务不是年付预付，客户要求月付、季付或后付。风险等级：中。处理建议：评估现金流和坏账风险，建议争取年付预付或服务期前到账。Agent 使用要求：不得默认所有后付条件都可接受。

规则 FIN-011：服务开始前到账。适用场景：开通账号、启动实施、服务开始。触发条件：服务开通、账号交付或实施启动早于首笔款到账。风险等级：高。处理建议：建议改为首笔款到账后启动服务，或限制交付范围。Agent 使用要求：必须提示先交付后收款风险。

规则 FIN-012：首付覆盖实施成本。适用场景：实施成本与首付款匹配。触发条件：首付款金额低于预计实施、定制或迁移成本。风险等级：高。处理建议：建议提高首付或拆分交付，避免先投入后回款。Agent 使用要求：必须联动交付可行性 Agent。

规则 FIN-013：账期超过 60 天。适用场景：账期与回款风险判断。触发条件：付款周期超过 60 天。风险等级：中。处理建议：建议财务复核客户信用，缩短账期或提高首付。Agent 使用要求：未查客户信用时不得输出低风险。

规则 FIN-014：账期达到 90 天。适用场景：长账期付款。触发条件：付款周期达到或超过 90 天。风险等级：高。处理建议：建议升级复核，争取缩短至 60 天以内或增加担保条件。Agent 使用要求：必须提示回款周期和现金流风险。

规则 FIN-015：大额长账期。适用场景：大额合同叠加长账期。触发条件：金额大于等于 500000 且账期超过 60 天。风险等级：高。处理建议：输出组合风险，要求财务负责人复核。Agent 使用要求：不得仅按单项账期规则判断。

规则 FIN-016：发票后付款不清。适用场景：开票后付款条款。触发条件：付款条件为收到发票后 N 天付款，但未明确开票条件、付款起算日和逾期责任。风险等级：中。处理建议：建议明确开票、付款和逾期责任。Agent 使用要求：必须提示付款触发条件不清。

规则 FIN-017：超长账期例外。适用场景：超长账期、验收后再起算账期。触发条件：账期大于等于 120 天，或验收后再起算 90 天。风险等级：高。处理建议：标记为重大例外，建议管理层和财务共同确认。Agent 使用要求：不得输出常规推进结论。

规则 FIN-018：三期以上分期。适用场景：分期付款。触发条件：分三期及以上付款。风险等级：中。处理建议：明确每期金额、付款条件、触发节点和逾期处理。Agent 使用要求：不得输出模糊的分期可接受。

规则 FIN-019：尾款占比过高。适用场景：尾款、验收后付款。触发条件：最后一笔付款占比大于 30%，或验收后尾款大于 20%。风险等级：高。处理建议：建议降低尾款比例，提高上线前或阶段交付前回款比例。Agent 使用要求：必须提示尾款回收不确定性。

规则 FIN-020：分期节点不对价。适用场景：付款节点与交付物匹配。触发条件：付款节点与交付成果、服务周期或使用权释放不匹配。风险等级：中。处理建议：建议按交付物和服务期重新设计节点。Agent 使用要求：必须联动交付计划判断。

规则 FIN-021：验收后付款。适用场景：尾款或主要款项依赖客户验收。触发条件：尾款或主要款项依赖客户验收后支付。风险等级：高。处理建议：联动合同和交付 Agent，要求明确验收标准、材料、周期和默认验收。Agent 使用要求：不得在验收机制不清时输出可接受。

规则 FIN-022：验收后长账期。适用场景：验收后付款叠加长账期。触发条件：验收后付款且验收后账期超过 30 天。风险等级：高。处理建议：建议缩短验收后账期，或设置默认验收和逾期责任。Agent 使用要求：必须提示验收不确定和回款延迟叠加风险。

规则 FIN-023：信用必查。适用场景：账期、分期、低首付、验收后付款。触发条件：客户要求账期、分期、低首付或验收后付款。风险等级：中。处理建议：必须查询客户信用等级、历史回款和未清应收。Agent 使用要求：缺少信用信息时不得输出信用通过判断。

规则 FIN-024：新客户后付限制。适用场景：新客户首单付款风险。触发条件：新客户或无交易历史客户要求后付、长账期或低首付。风险等级：高。处理建议：建议首单采用预付、短账期或阶段交付。Agent 使用要求：不得把新客户后付视作常规付款条件。

规则 FIN-025：历史逾期限制。适用场景：历史欠款、逾期、坏账。触发条件：客户存在历史逾期、未清欠款或坏账记录。风险等级：高。处理建议：建议结清欠款或获得财务授权后再推进。Agent 使用要求：必须把逾期记录作为高风险依据。

规则 FIN-026：信用额度超限。适用场景：授信额度、未清应收叠加。触发条件：本次金额加未清应收超过客户信用额度。风险等级：高。处理建议：建议调整付款条件或要求额外担保。Agent 使用要求：不得输出授信通过结论。

规则 FIN-027：信息缺失禁止低风险。适用场景：关键字段缺失。触发条件：缺少金额、折扣、账期、首付、客户信用任一关键字段。风险等级：阻塞。处理建议：不得输出低风险或可接受结论，应列出缺失信息。Agent 使用要求：必须先追问或提示人工补充。

规则 FIN-028：续费降价。适用场景：续费报价、续约折扣。触发条件：续费报价低于上一合同单价、ARR 或授权折扣线。风险等级：中。处理建议：要求说明降价原因、流失风险、替代方案和管理层授权。Agent 使用要求：不得默认续费降价合理。

规则 FIN-029：欠款客户续费或增购。适用场景：欠款客户续费、增购、扩容。触发条件：客户存在未清欠款时申请续费、增购或扩容。风险等级：高。处理建议：建议先处理欠款，再评估新增交易。Agent 使用要求：必须提示欠款与新增交易冲突。

规则 FIN-030：增购沿用历史低价。适用场景：增购、扩容、历史特殊低价。触发条件：增购要求沿用历史特殊低价或低于当前价目表授权价。风险等级：中。处理建议：建议重新评估当前价格、服务期和客户价值。Agent 使用要求：不得自动延续历史低价。

规则 FIN-031：Co-term 收入损失。适用场景：增购合并原合同周期。触发条件：增购合并到原合同周期导致实际服务期拉长但收费不足。风险等级：中。处理建议：建议按剩余服务期折算，明确起止日期和收费口径。Agent 使用要求：必须提示收入确认和服务期风险。

规则 FIN-032：低折扣叠加长账期。适用场景：组合财务风险。触发条件：折扣低于 80 折，且账期超过 60 天或首付低于 50%。风险等级：高。处理建议：建议至少调整一个条件，提高折扣、缩短账期、提高首付或减少范围。Agent 使用要求：必须输出组合风险，而不是单点风险。

规则 FIN-033：高成本交付组合。适用场景：定制开发、数据迁移、紧急上线叠加付款风险。触发条件：包含定制开发、数据迁移、紧急上线，同时低首付或验收后付款。风险等级：高。处理建议：联动交付 Agent，要求成本覆盖和阶段付款。Agent 使用要求：必须同时考虑交付成本和回款节点。

规则 FIN-034：多中风险升级。适用场景：多个中风险叠加。触发条件：任意两个中风险同时存在。风险等级：高。处理建议：升级为综合财务风险，建议跨角色复核。Agent 使用要求：不得把多个中风险分别淡化处理。

规则 FIN-035：审批路径映射。适用场景：命中高风险后的审批路由。触发条件：命中折扣、金额、账期、信用、验收付款任一高风险。风险等级：高。处理建议：输出需要参与的审批角色和材料清单。Agent 使用要求：不得只给风险结论，必须给复核角色。

规则 FIN-036：审批材料完整性。适用场景：提交审批前检查。触发条件：需要审批但缺少客户价值、竞品报价、毛利影响、回款保障或替代方案。风险等级：阻塞。处理建议：先补充材料，不建议提交审批。Agent 使用要求：必须列出缺失材料。

规则 FIN-037：定价模型适配判断。适用场景：产品报价、定价方案选择。触发条件：商机涉及按用户数、阶梯、用量或固定订阅等多种计价方式，但未明确适用场景和计价口径。风险等级：中。处理建议：根据客户规模、使用场景和行业惯例判断最优定价模型，按用户数适合组织规模稳定的客户，阶梯定价适合成长型客户，用量计价适合波动型使用场景，固定订阅适合中小客户。Agent 使用要求：不得在未确认客户使用规模和场景时推荐单一定价模型。

规则 FIN-038：产品版本分级定价。适用场景：产品选型、版本报价。触发条件：商机涉及标准版、专业版、企业版等多个版本，但未明确功能边界和价格阶梯。风险等级：中。处理建议：确认各版本功能边界、用户数上限、存储额度、接口配额和服务级别差异，输出版本对比和价格阶梯建议。Agent 使用要求：不得在功能边界不清时输出版本推荐结论。

规则 FIN-039：底价设定与年度审核。适用场景：产品底价管理、折扣审批基准。触发条件：折扣审批涉及产品底价，但底价未经定价委员会审核或超过一年未更新。风险等级：高。处理建议：每产品线须设定绝对底价，由定价委员会至少每年审核一次，底价调整须同步更新审批矩阵和折扣授权线。Agent 使用要求：不得引用过期底价作为审批依据。

规则 FIN-040：定价变更通知规则。适用场景：产品价格调整、价目表更新。触发条件：产品价格发生调整，但未提前通知客户或通知内容不完整。风险等级：中。处理建议：价格调整须提前 30 天书面通知客户，通知内容须包含变更范围、调整原因、新价格生效日期、客户影响和过渡方案。Agent 使用要求：不得在未确认通知是否已发送时输出价格已生效结论。

规则 FIN-041：定价优化审查周期。适用场景：定价策略迭代、价格竞争力评估。触发条件：产品定价超过一个季度未进行市场竞争力测试和定价迭代。风险等级：低。处理建议：建议至少每季度进行一次定价测试，评估竞品定价、客户接受度、转化率和利润率，输出定价优化建议。Agent 使用要求：定价审查结果仅作为内部参考，不直接输出给客户。

规则 FIN-042：续费价格计算基准。适用场景：续费报价、续约价格审批。触发条件：续费报价低于上一合同单价或 ARR，或续费时未明确价格计算基准。风险等级：中。处理建议：续费默认按原合同价格执行，涨价须说明依据如产品升级、服务范围扩大或市场价调整，并提前与客户沟通。Agent 使用要求：不得默认续费降价合理，也不得在未沟通情况下输出涨价建议。

规则 FIN-043：续费锁价期规则。适用场景：多年合同续费、锁价期到期续约。触发条件：多年合同锁价期内续费报价高于约定价格，或锁价期即将到期但未启动续约谈判。风险等级：高。处理建议：锁价期内续费不得高于合同约定的锁价价格，锁价期到期前 90 天须启动续约谈判，评估新周期价格策略。Agent 使用要求：必须核对原合同锁价条款，不得忽略锁价约定。

规则 FIN-044：Co-term 折算公式。适用场景：增购合并原合同周期、对齐服务期。触发条件：增购要求合并到原合同周期，导致实际服务期拉长但收费未按比例折算。风险等级：中。处理建议：增购合并原合同时按剩余服务期天数比例折算费用，折算公式为增购年费乘以剩余天数除以 365，须明确起止日期和收费口径。Agent 使用要求：必须输出折算计算过程，不得直接给出总价。

规则 FIN-045：增购席位计价口径。适用场景：增购用户席位、扩容授权。触发条件：增购席位要求沿用历史特殊低价或低于当前价目表授权价。风险等级：中。处理建议：增购席位须按当前价目表价格计算，不得自动延续历史特殊低价；历史低价须经审批确认是否适用于新增购。Agent 使用要求：必须核对当前价目表和历史合同价格差异。

规则 FIN-046：多年期合同折扣梯度。适用场景：多年合同报价、合同周期谈判。触发条件：客户要求多年期合同但折扣未按梯度设定。风险等级：中。处理建议：2 年合同可额外给予 3-5% 折扣，3 年合同可额外给予 5-10% 折扣，多年合同须设置锁价条款和年度涨价上限。Agent 使用要求：不得在未评估收入确认影响时输出多年合同折扣建议。

规则 FIN-047：续费降级风险判断。适用场景：续费时减席位、降版本。触发条件：续费时客户要求减少席位数量或降低产品版本。风险等级：高。处理建议：评估降级对 ARR 的影响、替代方案如缩减范围而非降版本、流失风险和客户实际使用情况，输出降级影响分析和挽留建议。Agent 使用要求：不得默认续费降级可接受，必须输出收入影响和替代方案。

规则 FIN-048：实施费定价模型选择。适用场景：实施费报价、专业服务定价。触发条件：商机涉及实施服务但未明确定价模型。风险等级：中。处理建议：根据项目确定性选择定价模型，固定价格适合范围明确的项目，时间和材料适合需求不确定的项目，里程碑定价适合大型分阶段项目。Agent 使用要求：不得在交付范围未确认时推荐固定价格模型。

规则 FIN-049：实施成本覆盖线。适用场景：实施费评估、交付成本覆盖判断。触发条件：实施费低于订阅年费的 15% 或低于预计交付成本。风险等级：高。处理建议：标准交付实施费不低于订阅年费的 15-25%，含定制的交付须按工时核算不低于成本线，减免实施费须同步缩减交付范围。Agent 使用要求：必须联动交付 Agent 评估实施成本。

规则 FIN-050：定制开发人天费率基准。适用场景：定制开发报价、人天费率审批。触发条件：定制开发报价涉及人天费率，但费率未经审核或低于成本线。风险等级：中。处理建议：按顾问级别设定人天单价基准，初级顾问不低于市场费率下限，高级顾问和架构师按溢价费率计算，费率须每年审核更新。Agent 使用要求：不得引用未经审核的人天费率。

规则 FIN-051：专业服务利润率红线。适用场景：专业服务报价、利润率审批。触发条件：专业服务报价利润率低于 15%。风险等级：高。处理建议：专业服务目标利润率为 15-25%，低于 15% 须审批并说明战略价值，亏本服务须有明确的交叉销售或续费回报预期。Agent 使用要求：不得在未评估利润率时输出专业服务报价建议。

规则 FIN-052：实施费减免与范围缩减联动。适用场景：实施费减免、交付范围调整。触发条件：客户要求减免实施费但未缩减交付范围。风险等级：高。处理建议：减免实施费必须同步缩减交付范围，如减少培训场次、缩减数据迁移范围、降低定制开发工时或取消驻场服务。Agent 使用要求：必须输出费用减免与范围缩减的对应关系。

规则 FIN-053：数据迁移费定价规则。适用场景：数据迁移报价、历史数据导入定价。触发条件：商机涉及数据迁移服务但未按阶梯定价。风险等级：中。处理建议：数据迁移费按数据量、对象复杂度和迁移轮次阶梯计价，须明确数据对象类型、预估数据量、字段映射复杂度和验证轮次。Agent 使用要求：不得在数据范围未确认时输出固定迁移报价。

规则 FIN-054：价税分离条款。适用场景：合同报价、发票开具、税务合规判断。触发条件：合同或报价未列明不含税价款、适用税率、税额和价税合计。风险等级：高。处理建议：合同必须列明四要素即不含税价款、适用税率、税额和价税合计，长期合同须锁定不含税价，税额随税率政策浮动调整。Agent 使用要求：不得在价税分离条款缺失时输出低风险结论。

规则 FIN-055：增值税税率适用。适用场景：复合交易报价、税率分项判断。触发条件：交易同时涉及软件服务、货物销售和安装服务，但未分项列明适用税率。风险等级：高。处理建议：软件服务适用 6% 增值税税率，货物销售适用 13%，安装服务适用 9%，复合交易须分项列明各项价款和适用税率，不得混合按单一税率开票。Agent 使用要求：必须提示税率分项要求，不得默认全部按 6% 处理。

规则 FIN-056：发票类型与抵扣。适用场景：发票开具、客户抵扣需求确认。触发条件：未确认客户纳税人身份或发票类型不匹配。风险等级：中。处理建议：须确认客户纳税人身份，一般纳税人开具增值税专用发票，小规模纳税人开具普通发票，发票类型须在合同中约定并与客户确认。Agent 使用要求：不得在纳税人身份未确认时输出发票可开具结论。

规则 FIN-057：纳税人身份变更通知。适用场景：客户纳税人身份变更、合同税务条款调整。触发条件：客户纳税人身份发生变更如小规模转一般纳税人或反向变更但未通知。风险等级：中。处理建议：交易方纳税人身份变更须 3 日内书面通知，并签署补充协议调整发票类型和税率条款。Agent 使用要求：必须提示身份变更对发票和税率的影响。

规则 FIN-058：税率政策变动分担。适用场景：长期合同税率调整、政策变动。触发条件：国家税率政策发生调整，但合同未约定税率变动分担机制。风险等级：中。处理建议：长期合同须锁定不含税价，税额随税率政策浮动调整，合同须约定税率变动的通知机制和价格调整方式。Agent 使用要求：不得在税率政策变动时默认原税率仍适用。

规则 FIN-059：客户分级标准。适用场景：客户分级管理、差异化政策适用。触发条件：大额商机或战略客户商机缺少客户分级标识。风险等级：中。处理建议：按 ARR 规模、战略价值和行业标杆属性将客户分为 A/B/C 三级，A 级为年费 100 万以上或行业标杆客户，B 级为年费 30-100 万的成长客户，C 级为年费 30 万以下的标准客户。Agent 使用要求：不得在客户分级缺失时输出战略折扣建议。

规则 FIN-060：分级折扣授权差异。适用场景：不同客户等级的折扣授权。触发条件：折扣审批未区分客户等级，B/C 级客户申请 A 级折扣。风险等级：高。处理建议：A 级客户可申请战略折扣但须附战略价值评估材料，B/C 级客户按标准授权线执行，不得跨级授权。Agent 使用要求：必须核对客户分级与折扣授权线是否匹配。

规则 FIN-061：战略客户材料要求。适用场景：战略客户折扣审批、标杆客户认定。触发条件：以战略客户或标杆客户为折扣理由但缺少可验证材料。风险等级：中。处理建议：战略折扣须附可验证的业务价值如 ARR 贡献、标杆价值如案例授权、复购潜力如多产品线规划和成交概率评估。Agent 使用要求：不得仅凭客户名称或行业地位输出战略折扣建议。

规则 FIN-062：多产品捆绑报价。适用场景：多产品组合销售、捆绑折扣审批。触发条件：商机涉及多产品捆绑销售但折扣未按捆绑规则设定。风险等级：中。处理建议：捆绑销售可额外给予 3-5% 折扣，但不得低于各产品底价加权平均值，须评估捆绑对收入确认和单独售价分摊的影响。Agent 使用要求：必须输出各产品单独报价和捆绑后价格对比。

规则 FIN-063：独立与捆绑合同选择。适用场景：多产品合同结构选择。触发条件：多产品可签独立合同或统一合同但未评估影响。风险等级：中。处理建议：多产品可签独立合同或统一合同，须评估收入确认影响、终止条款联动、续费灵活性、折扣差异和合同管理复杂度。Agent 使用要求：不得在未评估收入确认影响时输出合同结构建议。

规则 FIN-064：外币报价与汇率风险。适用场景：外币报价、跨境交易。触发条件：外币报价未锁定汇率有效期或超期未重新报价。风险等级：高。处理建议：外币报价须锁定汇率有效期通常 30 天，超期须按新汇率重新报价，长期合同须约定汇率波动超过阈值时的调整机制。Agent 使用要求：必须提示汇率风险和报价有效期。

规则 FIN-065：跨境付款方式。适用场景：境外客户付款、跨境结算。触发条件：境外客户付款但未确认外汇合规、代扣代缴税和结算路径。风险等级：高。处理建议：境外客户付款须确认外汇合规要求、代扣代缴增值税和企业所得税、结算币种和路径、银行手续费承担方。Agent 使用要求：必须联动法务和财务评估跨境合规风险。

输出要求：财务风控 Agent 输出必须包含结论、风险等级、命中规则、财务风险点、建议调整方案、人工确认节点和 CRM 写回草稿。结论取值包括可常规推进、有条件推进、建议升级复核、暂不建议推进、信息不足。CRM 写回草稿只记录风险判断和待确认事项，不写审批通过结论。

禁用边界：财务风控 Agent 不得输出“折扣已批准”“账期可以承诺”“财务已通过”；不得在缺少客户信用和历史回款信息时输出低风险；不得生成可直接发给客户的正式报价承诺；不得用客户重要、老板要求或战略客户替代财务判断依据；不得忽略交付成本、合同验收和客户信用对回款的影响。

## 02 交付可行性规则

以下内容来源于 02-交付与定制规则，用于交付可行性 Agent。

文档说明：本文档用于支撑交付可行性 Agent 判断上线周期、实施范围、定制开发、数据迁移、系统集成、验收、客户资源和上线风险。规则输出应作为内部交付评估建议，不代表交付团队已承诺排期或资源。适用任务包括上线周期评估、标准交付判断、定制开发评估、接口集成评估、数据迁移评估、验收风险判断、客户资源风险判断和 Deal Review Brief 中的交付结论生成。不适用任务包括替代项目经理确认正式排期、自动承诺上线日期、替代技术方案评审、自动接受客户新增范围。

关键输入字段：交付可行性 Agent 应优先读取项目范围、客户期望上线时间、部署形态、定制范围、接口集成范围、数据迁移范围、客户资源、验收标准、培训需求和变更需求。若项目范围缺失，不得判断标准或非标；若上线时间缺失，只能输出初步计划；若部署形态缺失，无法判断环境和合规风险；若定制、集成、迁移范围缺失，不得承诺对应周期；若客户资源或验收标准缺失，应标记为高风险或阻塞。

风险等级定义：低风险表示标准范围、资源明确、周期合理，可给出计划建议；中风险表示存在局部不确定性或轻度压缩周期，建议补充信息或交付负责人复核；高风险表示存在紧急上线、定制、集成、迁移、验收或客户资源重大风险，建议升级复核，不得承诺日期；阻塞表示缺少关键范围、验收或资源信息，应先追问或要求人工确认。

规则 DEL-001：标准交付包边界确认。适用场景：标准交付、快速上线、开箱即用。触发条件：客户要求标准交付、快速上线或开箱即用，但未明确模块、配置、初始化数据、培训和上线支持范围。风险等级：中。处理建议：输出标准交付假设，要求确认范围边界。Agent 使用要求：不得把未确认范围默认为标准交付内容。

规则 DEL-002：两周内标准上线。适用场景：标准模块上线周期评估。触发条件：客户要求标准模块在 2 周内上线。风险等级：中。处理建议：确认客户资源、数据准备、培训和验收安排后再判断。Agent 使用要求：只能输出有条件可行，不得直接承诺固定上线日期。

规则 DEL-003：两周内全量上线。适用场景：紧急上线、全量上线。触发条件：客户要求两周内完成全量上线，且包含多个模块、复杂配置或多团队协作。风险等级：高。处理建议：建议拆分一期标准上线、二期优化或定制。Agent 使用要求：必须提示高风险，不得输出可以保证全量上线。

规则 DEL-004：固定上线日期。适用场景：客户指定不可调整 deadline。触发条件：客户要求必须在某日期上线且不可调整。风险等级：高。处理建议：要求交付负责人确认资源和客户配合，输出排期假设。Agent 使用要求：不得替交付团队承诺排期。

规则 DEL-005：资源未锁定。适用场景：资源排期评估。触发条件：实施顾问、技术支持、客户项目经理、客户 IT 或业务负责人未确认。风险等级：高。处理建议：只能输出计划建议，不得承诺日期。Agent 使用要求：输出中必须说明资源确认是上线承诺的前置条件。

规则 DEL-006：标准交付前置条件不足。适用场景：标准上线准入判断。触发条件：客户环境、账号、基础数据、管理员、业务负责人、上线范围任一未确认。风险等级：阻塞。处理建议：列出前置条件清单，要求补充后再评估。Agent 使用要求：信息不足时不得输出低风险或可上线结论。

规则 DEL-007：云端部署数据合规。适用场景：Dify Cloud、SaaS 云端服务、外部模型调用。触发条件：项目使用云端服务或外部模型，且涉及真实客户、商机、合同数据。风险等级：高。处理建议：建议脱敏、最小化传输，并确认客户数据授权和安全要求。Agent 使用要求：必须联动合同或安全复核，不得忽略数据路径。

规则 DEL-008：私有化部署评估。适用场景：私有化、本地化、专有云、内网、信创环境。触发条件：客户要求私有化、本地化、专有云、内网部署、信创环境或不得出网。风险等级：高。处理建议：单独评估部署架构、环境、运维、升级和成本。Agent 使用要求：不得按普通 SaaS 标准交付周期判断。

规则 DEL-009：客户安全审查。适用场景：安全准入、合规审查。触发条件：客户要求等保、渗透测试、安全测评、供应商准入或数据不出境。风险等级：中。处理建议：联动法务和安全负责人，输出所需材料清单。Agent 使用要求：不得承诺安全审查周期。

规则 DEL-010：生产数据脱敏。适用场景：测试、演示、知识库、外部模型调用。触发条件：测试、演示、Dify 调用或第三方工具中使用生产敏感数据。风险等级：高。处理建议：建议使用脱敏数据或样例数据，不得直接传输敏感信息。Agent 使用要求：必须提示数据最小化和脱敏要求。

规则 DEL-011：数据迁移范围确认。适用场景：历史数据导入、初始化数据。触发条件：客户要求导入历史客户、联系人、商机、合同、跟进记录、产品、权限或其他存量数据。风险等级：中。处理建议：明确数据对象、字段、数据量、模板和验收口径。Agent 使用要求：缺少迁移范围时不得承诺迁移周期。

规则 DEL-012：数据质量客户责任。适用场景：数据清洗、迁移前准备。触发条件：客户无法提供标准模板、字段说明、样例数据、清洗负责人或验收人。风险等级：高。处理建议：标记客户配合风险，建议客户先完成数据清洗。Agent 使用要求：必须说明客户数据质量会影响上线周期。

规则 DEL-013：多轮迁移验证。适用场景：大批量历史数据迁移、复杂字段映射。触发条件：数据量大、字段复杂、存在历史脏数据，或上线前要求全量迁移。风险等级：高。处理建议：建议设置试迁移、校验、修正、正式迁移和增量切换。Agent 使用要求：不得把全量迁移视为一次性简单导入。

规则 DEL-014：数据冻结与切换。适用场景：旧系统切换、新旧系统并行。触发条件：正式上线涉及旧系统切换或增量数据同步。风险等级：高。处理建议：要求明确数据冻结时间、回滚方案和责任人。Agent 使用要求：必须提示上线切换和回滚风险。

规则 DEL-015：非标接口准入。适用场景：ERP、WMS、OMS、财务、SSO、审批流、短信、邮件、BI 或第三方平台集成。触发条件：客户要求对接任一外部系统。风险等级：高。处理建议：要求接口清单、协议、认证、字段映射、测试环境和联调计划。Agent 使用要求：未提供接口资料时不得承诺联调周期。

规则 DEL-016：第三方接口依赖。适用场景：客户 IT 或第三方厂商参与联调。触发条件：集成依赖客户 IT 或第三方厂商提供接口、账号、密钥、测试数据或问题响应。风险等级：高。处理建议：在计划和合同中明确第三方延期不由我方承担。Agent 使用要求：必须提示第三方响应会影响交付周期。

规则 DEL-017：接口变更重估。适用场景：已确认接口后的变更。触发条件：客户或第三方调整字段、协议、认证方式、业务逻辑或调用频率。风险等级：高。处理建议：触发变更评估，不得沿用原排期。Agent 使用要求：必须提示范围、周期和费用可能变化。

规则 DEL-018：SSO 和权限集成。适用场景：单点登录、组织架构同步、角色权限、审计日志。触发条件：项目涉及 SSO、组织同步、权限矩阵或审计日志要求。风险等级：中。处理建议：要求安全评审、测试账号和权限矩阵。Agent 使用要求：不得仅按普通配置项处理。

规则 DEL-019：定制需求范围确认。适用场景：定制看板、报表、流程、字段逻辑、权限、页面或算法规则。触发条件：客户提出任一非标准定制诉求。风险等级：中。处理建议：要求需求范围、原型、字段、数据源、验收样例。Agent 使用要求：不得把定制需求当作标准功能承诺。

规则 DEL-020：定制技术评审。适用场景：复杂定制、核心流程改造。触发条件：定制涉及核心流程、权限模型、复杂计算、跨系统数据、性能或安全要求。风险等级：高。处理建议：先做技术方案评审，再输出周期和报价。Agent 使用要求：不得直接给出确定开发周期。

规则 DEL-021：标准加定制混合交付。适用场景：标准模块与定制需求混合上线。触发条件：项目同时包含标准模块和定制需求。风险等级：中。处理建议：建议一期标准上线，二期评估和交付定制。Agent 使用要求：输出中应明确一期和二期范围。

规则 DEL-022：范围变更评估。适用场景：新增、删除或调整需求。触发条件：客户新增、删除或调整模块、字段、流程、报表、接口、数据范围、权限或验收标准。风险等级：中。处理建议：输出变更影响，包括工作量、周期、费用、验收和合同。Agent 使用要求：不得默认为免费或不影响排期。

规则 DEL-023：变更未确认不得开发。适用场景：变更单、补充协议或邮件确认。触发条件：变更影响工作量、周期、费用、架构或验收，但未完成正式确认。风险等级：高。处理建议：不建议进入开发，先完成变更确认。Agent 使用要求：必须提示人工确认节点。

规则 DEL-024：UAT 准入。适用场景：客户验收测试。触发条件：客户要求进入 UAT，但配置、数据、接口、培训、测试账号或测试用例未准备完成。风险等级：中。处理建议：输出 UAT 准入清单和阻塞项。Agent 使用要求：不得把未满足准入条件的项目判断为可验收。

规则 DEL-025：UAT 周期压缩。适用场景：测试周期缩短、缺陷处理标准缺失。触发条件：客户要求压缩 UAT 周期，或未定义 P0、P1、P2 缺陷标准。风险等级：高。处理建议：明确缺陷分级、修复时限和是否阻断上线。Agent 使用要求：必须提示质量和上线风险。

规则 DEL-026：验收标准缺失。适用场景：验收付款、上线验收、阶段验收。触发条件：项目要求验收付款、上线验收或阶段验收，但验收标准、材料、签署人、反馈期限缺失。风险等级：高。处理建议：联动合同 Agent，要求补充验收机制。Agent 使用要求：不得在验收标准不清时承诺固定上线或回款节点。

规则 DEL-027：默认验收缺失。适用场景：客户长期不反馈验收。触发条件：客户可能长期不反馈 UAT 或验收结果，合同或计划未约定默认验收期限。风险等级：高。处理建议：建议设置逾期未反馈视为验收通过。Agent 使用要求：必须联动合同风险判断。

规则 DEL-028：上线 Go/No-Go。适用场景：生产上线前检查。触发条件：生产上线前未完成上线清单、回滚方案、数据冻结、权限确认或支持窗口确认。风险等级：高。处理建议：不建议承诺上线，先完成上线检查。Agent 使用要求：必须输出上线前置清单。

规则 DEL-029：上线后保障期。适用场景：驻场、陪跑、快速响应、7x24 支持。触发条件：客户要求上线后保障，但未明确周期、响应级别和费用。风险等级：中。处理建议：明确保障范围和服务边界。Agent 使用要求：不得默认包含额外支持服务。

规则 DEL-030：客户资源配合。适用场景：客户项目经理、Key User、IT、数据负责人、验收人协作。触发条件：任一关键客户角色未指定或无法承诺响应时效。风险等级：高。处理建议：输出客户侧资源清单和响应要求。Agent 使用要求：必须说明客户资源是交付成功条件。

规则 DEL-031：延期赔付联动。适用场景：固定上线承诺、延期赔付、服务可用性赔付。触发条件：合同包含延期赔付、服务可用性赔付或固定上线承诺，且交付范围或客户责任不清。风险等级：高。处理建议：联动合同 Agent，建议明确免责和责任边界。Agent 使用要求：必须提示合同与交付联动风险。

规则 DEL-032：客户原因延期免责。适用场景：客户配合导致延期。触发条件：延期由客户数据、接口、验收、人员、权限、安全审批、环境准备导致。风险等级：高。处理建议：建议合同和项目计划中明确顺延机制。Agent 使用要求：不得把客户原因延期计入我方无条件责任。

规则 DEL-033：分阶段交付。适用场景：一期、二期、多阶段上线。触发条件：项目被拆成一期、二期或多阶段交付。风险等级：中。处理建议：明确每阶段范围、验收标准、付款节点和依赖关系。Agent 使用要求：输出中应区分阶段目标。

规则 DEL-034：阶段间依赖。适用场景：二期定制依赖一期结果。触发条件：二期定制依赖一期数据、流程、接口或客户反馈，且客户要求提前锁定二期上线日期。风险等级：高。处理建议：输出依赖条件，不建议提前承诺固定日期。Agent 使用要求：不得在依赖未满足时承诺二期排期。

规则 DEL-035：交付口径不一致。适用场景：销售、售前、交付、产品口径冲突。触发条件：不同角色对上线日期、范围、定制、赔付或验收口径不一致。风险等级：高。处理建议：建议内部对齐后再回复客户。Agent 使用要求：必须提示需要人工确认统一口径。

规则 DEL-036：等保二级/三级要求。适用场景：客户要求等保、安全测评。触发条件：客户要求系统满足等保二级或三级，但未确认系统定级和安全整改范围。风险等级：高。处理建议：须确认客户系统定级报告、安全整改范围、测评机构要求和整改周期，等保三级须评估物理安全、网络安全、主机安全、应用安全和数据安全等多层要求。Agent 使用要求：不得按普通 SaaS 交付周期判断等保项目，必须联动安全负责人评估。

规则 DEL-037：ISO 认证材料清单。适用场景：客户要求 ISO 认证、供应商准入。触发条件：客户要求供应商具备 ISO 27001 或 ISO 27701 认证。风险等级：中。处理建议：须提供认证证书复印件、认证范围说明、适用性声明和最近一次监督审核报告，确认认证范围覆盖客户使用的服务模块。Agent 使用要求：不得在认证范围未确认时输出已满足要求。

规则 DEL-038：金融行业特殊合规。适用场景：金融客户交付、银行保险证券。触发条件：客户为银行、保险、证券或其他持牌金融机构。风险等级：高。处理建议：金融客户须额外满足银保监会或证监会数据安全要求，包括数据本地化存储、核心系统不得外包、灾备要求、监管报送接口和审计留痕。Agent 使用要求：不得按普通企业标准交付金融客户项目。

规则 DEL-039：医疗行业数据合规。适用场景：医疗客户交付、健康数据。触发条件：客户为医疗机构或涉及健康医疗数据。风险等级：高。处理建议：医疗客户须确认个人信息保护法合规要求、健康数据特殊保护要求、医疗数据分类分级和医疗数据不得出境要求。Agent 使用要求：必须联动法务评估医疗行业数据合规风险。

规则 DEL-040：政府和国企信创要求。适用场景：政府客户、国企客户、信创环境。触发条件：客户为政府机关或国有企业，要求信创适配或国产化替代。风险等级：高。处理建议：须确认信创目录适配要求、国产化替代范围包括操作系统、数据库、中间件和芯片适配，以及信创环境下的部署架构和性能影响。Agent 使用要求：不得按普通 SaaS 部署方案判断信创项目。

规则 DEL-041：涉密系统处理边界。适用场景：涉密系统项目、保密资质。触发条件：客户项目涉及国家秘密或涉密信息系统。风险等级：高。处理建议：涉密系统项目须确认供应商保密资质等级，不得按普通项目处理，须由具备涉密资质的团队在涉密环境中实施。Agent 使用要求：不得在保密资质未确认时输出可交付结论。

规则 DEL-042：标准交付工时基准。适用场景：标准交付包范围、工时管控。触发条件：标准交付包未明确含工时上限或超出标准工时但未按定制计费。风险等级：中。处理建议：标准交付包须明确含工时上限，超出部分按定制开发人天费率计费，标准工时须按交付模块和客户规模分档设定。Agent 使用要求：不得在工时上限未确认时输出标准交付可覆盖所有需求。

规则 DEL-043：定制开发范围冻结。适用场景：定制需求确认、范围锁定。触发条件：定制需求未完成 PRD 和技术评审即要求锁定范围和报价。风险等级：高。处理建议：定制需求须完成 PRD 文档、技术方案评审和工时评估后方可锁定范围和报价，范围冻结后变更须走变更流程。Agent 使用要求：不得在 PRD 未确认时输出定制开发报价。

规则 DEL-044：交付范围变更工时影响。适用场景：需求变更、范围调整。触发条件：交付过程中客户提出范围变更，但未评估工时增量和费用调整。风险等级：中。处理建议：变更须评估工时增量、费用调整、排期影响和验收标准变化，输出变更影响评估表供客户确认。Agent 使用要求：不得默认变更不影响排期或费用。

规则 DEL-045：紧急交付加急费规则。适用场景：客户要求缩短交付周期、加急上线。触发条件：客户要求缩短标准交付周期但未评估加急成本和资源调配。风险等级：高。处理建议：加急交付须评估加急成本包括加班费、资源调配费和优先级调整费，加急费通常为标准交付费的 20-50%，须确认资源可行性和质量风险。Agent 使用要求：不得在未评估加急成本时输出可加速交付。

规则 DEL-046：SLA 可承诺基线。适用场景：SLA 承诺、可用性谈判。触发条件：合同涉及 SLA 承诺但未确认产品实际能力基线。风险等级：高。处理建议：标准版 SLA 可承诺 99.5% 可用性，企业版可承诺 99.9%，不得超出产品运维实际能力承诺，99.99% 以上须运维负责人确认。Agent 使用要求：不得替运维团队承诺超出能力的 SLA。

规则 DEL-047：保障期服务边界。适用场景：上线后保障期、免费支持范围。触发条件：上线后保障期未明确服务边界和周期。风险等级：中。处理建议：上线后保障期默认 30 天，保障范围含 bug 修复和配置调整，不含新需求开发和定制变更，超出保障期须按运维服务合同执行。Agent 使用要求：不得默认保障期包含新需求开发或无限期免费支持。

规则 DEL-048：延期赔付与交付联动。适用场景：固定上线日期承诺、延期赔付条款。触发条件：合同包含固定上线日期承诺和延期赔付条款，但交付方案未联动评估。风险等级：高。处理建议：固定上线日期承诺须联动合同延期赔付条款评估，确认赔付上限、免责条件和客户配合义务，交付方案须预留缓冲期。Agent 使用要求：必须联动合同 Agent 评估延期赔付风险。

输出要求：交付可行性 Agent 输出必须包含结论、风险等级、命中规则、周期与范围判断、客户前置条件、建议交付方案和 CRM 写回草稿。结论取值包括可按标准交付、有条件可行、需拆分范围、建议交付复核、暂不建议承诺。CRM 写回草稿只记录计划建议和待确认事项，不写正式交付承诺。

禁用边界：交付可行性 Agent 不得输出“保证两周上线”“交付已确认排期”“可以承诺客户日期”；不得在资源未锁定、范围未确认、验收未明确时输出确定性排期；不得将定制需求当作标准功能承诺；不得忽略第三方接口、客户数据、客户 IT 和验收人的配合责任；不得自动接受范围变更或新增开发。

## 03 合同与验收规则

以下内容来源于 03-合同与验收规则，用于合同风险 Agent。

文档说明：本文档用于支撑合同风险 Agent 识别 B 端 SaaS 商机中的合同条款、验收、付款绑定、延期赔付、责任上限、数据安全、知识产权和客户模板风险。输出仅作为内部合同风险识别和复核建议，不构成法律意见。适用任务包括合同条款审查、验收与付款绑定判断、延期赔付判断、SLA 风险判断、责任边界判断、数据安全条款判断、知识产权归属判断和 Deal Review Brief 中的合同风险结论生成。不适用任务包括替代法务出具正式法律意见、自动判断合同可签署、生成最终合同红线稿、承诺客户条款可接受。

关键输入字段：合同风险 Agent 应优先读取合同正文、订单、附件、客户模板条款、验收条款、付款条款、上线日期和延期赔付条款、SLA 条款、责任上限条款、数据安全条款、知识产权条款、解除和暂停服务条款、争议解决条款。若合同正文、附件或客户模板缺失，不得输出低风险或合同可签结论。

风险等级定义：低风险表示条款接近标准口径且未发现明显偏离，可提示常规关注点；中风险表示存在局部条款偏离或信息不足，建议法务或相关角色复核；高风险表示存在赔付、无限责任、验收不清、数据安全或知识产权高风险，必须升级法务复核；阻塞表示合同文本不足或用户要求最终法律结论，应先追问或声明边界。

规则 LEG-001：验收标准缺失。适用场景：验收付款、上线验收、阶段验收。触发条件：合同提到验收，但未列明交付物清单、验收材料、验收指标、测试环境或验收负责人。风险等级：高。处理建议：补充验收清单、材料、期限、签署人和异议处理机制。Agent 使用要求：不得接受验收后付款的低风险结论。

规则 LEG-002：默认验收缺失。适用场景：客户验收反馈期限。触发条件：合同要求客户验收，但未约定客户在 N 日内反馈异议、逾期视为验收通过。风险等级：高。处理建议：建议加入默认验收条款。Agent 使用要求：必须提示尾款和项目关闭不确定性。

规则 LEG-003：单方验收权。适用场景：客户单方验收。触发条件：客户单方决定是否验收通过，供应商缺少争议处理或默认验收机制。风险等级：高。处理建议：建议加入双方确认、客观标准和异议期限。Agent 使用要求：不得输出该条款可直接接受。

规则 LEG-004：客户配合义务缺失。适用场景：验收、上线、付款依赖客户配合。触发条件：验收、上线或付款依赖客户账号、数据、接口、人员、环境，但合同未写客户延误顺延机制。风险等级：高。处理建议：明确客户配合义务和顺延机制。Agent 使用要求：必须联动交付 Agent 判断客户资源风险。

规则 LEG-005：实际使用视同验收。适用场景：试运行、生产上线、客户实际使用。触发条件：客户试运行、上线生产或开放给终端用户，但合同未约定是否构成阶段验收。风险等级：中。处理建议：建议加入上线、试运行或实际使用视同验收规则。Agent 使用要求：提示需法务确认表达口径。

规则 LEG-006：付款完全依赖主观确认。适用场景：主观验收、客户满意、内部审批付款。触发条件：付款条件为客户满意、最终确认、内部审批完成等不可控主观条件。风险等级：高。处理建议：建议改为客观交付物、时间节点或默认验收触发。Agent 使用要求：必须提示回款不确定性。

规则 LEG-007：开票与付款倒挂。适用场景：先开票后付款。触发条件：客户要求先开全额发票再付款，或付款以收到发票和内部流程为唯一条件且无期限。风险等级：中。处理建议：明确开票条件、付款期限和逾期责任。Agent 使用要求：联动财务 Agent 判断回款风险。

规则 LEG-008：逾期付款缺少暂停权。适用场景：逾期付款、暂停服务权。触发条件：客户延期付款，但合同未约定暂停服务、暂停交付、违约金或解除权。风险等级：高。处理建议：补充逾期付款处理和暂停服务权。Agent 使用要求：不得忽略客户违约后的救济机制。

规则 LEG-009：延期赔付条款。适用场景：延期上线、逾期交付、违约金。触发条件：合同包含延期赔付、逾期赔偿、违约金或上线日期赔付。风险等级：高。处理建议：明确赔付上限、免责条件、客户配合义务和唯一救济。Agent 使用要求：必须联动交付 Agent 评估排期风险。

规则 LEG-010：延期赔付无上限。适用场景：赔付封顶。触发条件：延期赔付未按合同金额或阶段金额设置累计上限。风险等级：高。处理建议：建议设置累计赔付上限。Agent 使用要求：不得输出可接受结论。

规则 LEG-011：客户原因未排除。适用场景：延期责任免责。触发条件：延期责任未排除客户资料延迟、需求变更、接口延迟、第三方服务或不可抗力。风险等级：高。处理建议：建议加入客户原因和第三方原因免责。Agent 使用要求：必须提示责任边界不清。

规则 LEG-012：SLA 超出能力。适用场景：可用性承诺、7x24 响应、恢复时间。触发条件：合同要求 99.9%、99.99% 可用性、7x24 响应或固定恢复时间，但产品或运维能力未确认。风险等级：高。处理建议：联动交付和运维确认服务能力。Agent 使用要求：不得替运维团队承诺 SLA。

规则 LEG-013：SLA 补偿无唯一救济。适用场景：服务补偿、额外赔偿。触发条件：SLA 未达标既要求服务抵扣又保留额外赔偿或解除权，或补偿无上限。风险等级：高。处理建议：建议设置服务补偿为唯一救济，并明确上限。Agent 使用要求：必须提示赔偿暴露风险。

规则 LEG-014：维护窗口和免责缺失。适用场景：SLA 免责。触发条件：SLA 未排除计划维护、客户网络、第三方云服务、不可抗力、客户误操作、安全攻击等场景。风险等级：中。处理建议：补充维护窗口和免责场景。Agent 使用要求：不得把全部不可用原因归责供应商。

规则 LEG-015：责任上限缺失。适用场景：责任限制、赔偿上限。触发条件：合同没有责任上限，或责任上限超过已付金额、年度服务费或合同总额。风险等级：高。处理建议：建议设置合理责任上限。Agent 使用要求：必须建议法务复核。

规则 LEG-016：间接损失未排除。适用场景：利润损失、商誉损失、业务中断。触发条件：合同要求赔偿利润损失、商誉损失、数据损失、业务中断、替代采购等间接或特殊损失。风险等级：高。处理建议：建议排除间接损失和不可预见损失。Agent 使用要求：不得输出条款可接受。

规则 LEG-017：责任上限例外过宽。适用场景：责任上限例外。触发条件：客户将保密、数据、知识产权、合规、故意过失全部排除在责任上限之外。风险等级：高。处理建议：建议缩小例外范围并明确适用条件。Agent 使用要求：必须提示例外范围过宽。

规则 LEG-018：无限责任。适用场景：全部损失、无限责任、监管处罚。触发条件：出现承担全部损失、无限责任、所有监管处罚由供应商承担等表述。风险等级：高。处理建议：必须法务复核，不得直接接受。Agent 使用要求：不得给出可签署或可接受结论。

规则 LEG-019：数据处理角色不清。适用场景：客户数据、个人信息处理。触发条件：涉及客户数据或个人信息，但未明确双方处理角色、处理目的、范围和期限。风险等级：高。处理建议：明确处理角色、授权范围、保密和安全措施。Agent 使用要求：必须提示数据合规风险。

规则 LEG-020：数据泄露通知过严。适用场景：数据泄露、监管处罚。触发条件：合同要求立即或数小时内通知、承担全部损失、负责全部监管处罚。风险等级：高。处理建议：建议改为知悉后合理期限内通知，并限定责任边界。Agent 使用要求：不得接受绝对化安全责任。

规则 LEG-021：数据返还删除缺失。适用场景：合同终止后的数据处理。触发条件：合同终止后未约定客户数据导出、返还、删除、备份保留期限和例外。风险等级：中。处理建议：补充终止后的数据处理机制。Agent 使用要求：提示需明确保留和删除口径。

规则 LEG-022：第三方处理未授权。适用场景：云服务、外部模型、分包商、第三方工具。触发条件：使用云服务、外部模型、分包商或第三方工具处理数据，但未约定客户授权、清单和变更通知。风险等级：高。处理建议：明确第三方清单、授权和安全责任。Agent 使用要求：必须联动交付部署形态判断。

规则 LEG-023：跨境或不得出网。适用场景：数据本地化、不得出网、跨境处理。触发条件：客户要求数据不得出网、不得跨境或必须本地化处理。风险等级：高。处理建议：联动交付 Agent 评估部署形态和数据路径。Agent 使用要求：不得按云端默认方案判断。

规则 LEG-024：合规承诺绝对化。适用场景：监管合规、客户制度承诺。触发条件：合同要求保证完全符合法律法规、监管要求或客户全部制度。风险等级：高。处理建议：建议限定适用法律、产品能力和客户自身责任。Agent 使用要求：不得接受无限合规承诺。

规则 LEG-025：标准产品 IP 转让。适用场景：平台、源代码、底层技术归属。触发条件：客户要求标准产品、平台、源代码、底层技术或通用能力归客户所有。风险等级：高。处理建议：明确标准产品和底层技术归供应商所有。Agent 使用要求：必须提示知识产权转让风险。

规则 LEG-026：定制成果复用权缺失。适用场景：定制开发、报表、接口、配置、模板。触发条件：定制成果未区分客户专有成果和供应商可复用通用能力。风险等级：中。处理建议：明确专有成果、通用能力和复用边界。Agent 使用要求：不得默认全部成果归客户所有。

规则 LEG-027：源代码交付超范围。适用场景：源代码、数据库结构、部署脚本、Prompt、算法、运维权限。触发条件：客户要求交付源代码、数据库结构、部署脚本、模型 Prompt、算法或运维权限。风险等级：高。处理建议：单独评估交付范围、费用和安全风险。Agent 使用要求：不得直接承诺交付。

规则 LEG-028：客户反馈归属不清。适用场景：需求建议、行业反馈、产品改进。触发条件：合同未说明客户反馈、优化建议、行业需求是否可用于产品改进。风险等级：中。处理建议：建议保留产品改进和通用能力复用权。Agent 使用要求：提示需明确复用边界。

规则 LEG-029：客户模板偏离标准条款。适用场景：客户采购合同、客户模板、招标条款。触发条件：使用客户模板或采购条款，且与公司标准条款在付款、责任、数据、IP、解除、管辖上不一致。风险等级：高。处理建议：标记为法务重点审查。Agent 使用要求：不得按公司标准合同直接判断。

规则 LEG-030：文件优先级不利。适用场景：合同文件优先级。触发条件：客户采购条款、订单、附件、招标文件优先于主合同或我方标准条款。风险等级：高。处理建议：建议调整文件优先级。Agent 使用要求：必须提示条款冲突风险。

规则 LEG-031：客户单方解除权过宽。适用场景：无因解除、随时解除、轻微违约解除。触发条件：客户可无因解除、随时解除或因轻微违约解除。风险等级：高。处理建议：补充已交付费用、最低服务期、整改期和补偿。Agent 使用要求：不得忽略收入确认和交付投入风险。

规则 LEG-032：供应商暂停或解除权缺失。适用场景：逾期付款、违规使用、超范围使用。触发条件：客户逾期付款、违规使用、侵害安全、超范围使用时，供应商无暂停服务或解除权。风险等级：中。处理建议：补充暂停服务和解除权。Agent 使用要求：必须提示救济机制不足。

规则 LEG-033：自动续费不清。适用场景：自动续约、续费通知。触发条件：合同包含自动续费，但未明确续费周期、价格、提前通知、取消窗口和确认机制。风险等级：中。处理建议：明确续费规则和通知机制。Agent 使用要求：提示需明确续费口径。

规则 LEG-034：客户审计权过宽。适用场景：现场审计、渗透测试、源码查看。触发条件：客户可随时现场审计、渗透测试、查看系统、源码或其他客户信息。风险等级：高。处理建议：限制频次、范围、保密、费用和安全边界。Agent 使用要求：不得接受无限审计权。

规则 LEG-035：管辖争议不利。适用场景：管辖地、仲裁、适用法律。触发条件：管辖地为客户所在地、境外法院或仲裁，或适用法、语言不明确。风险等级：中。处理建议：建议法务评估争议解决成本。Agent 使用要求：不得输出最终法律判断。

规则 LEG-036：标准合同结构基准。适用场景：新建合同、合同结构审查。触发条件：合同缺少 MSA、AUP、SLA 或隐私政策任一标准文件。风险等级：中。处理建议：标准合同结构应包含 MSA 主服务协议、AUP 可接受使用政策、SLA 服务水平协议和隐私政策四件套，客户模板合同须逐条对标审查偏离项。Agent 使用要求：不得在合同结构不完整时输出合同可签结论。

规则 LEG-037：标准付款条款基准。适用场景：付款条款谈判、付款方案审批。触发条件：付款条款偏离标准付款基准。风险等级：中。处理建议：标准付款条款为年付预付，首付不低于 50%，账期不超过 60 天，付费方式为银行转账，偏离标准须审批并说明理由。Agent 使用要求：必须对标标准付款基准输出偏离项。

规则 LEG-038：标准责任上限基准。适用场景：责任上限条款谈判。触发条件：责任上限条款偏离标准基准。风险等级：高。处理建议：标准责任上限为已付年度服务费金额，须排除间接损失、利润损失和商誉损失，超过此基准须法务审批。Agent 使用要求：不得在责任上限偏离标准时输出条款可接受。

规则 LEG-039：标准验收条款基准。适用场景：验收条款审查、验收流程设计。触发条件：验收条款偏离标准基准。风险等级：中。处理建议：标准验收条款为交付后 15 个工作日内反馈，逾期视为验收通过，争议由双方协商解决，偏离标准须审批。Agent 使用要求：必须对标标准验收基准输出偏离项。

规则 LEG-040：标准 SLA 条款基准。适用场景：SLA 条款审查、服务水平承诺。触发条件：SLA 条款偏离标准基准。风险等级：高。处理建议：标准 SLA 为 99.9% 可用性、P1 事件 4 小时首次响应、服务抵扣为唯一救济且抵扣上限为月度服务费，偏离标准须运维和法务复核。Agent 使用要求：不得在 SLA 偏离标准时输出条款可接受。

规则 LEG-041：标准数据条款基准。适用场景：数据条款审查、数据安全条款。触发条件：数据条款偏离标准基准。风险等级：中。处理建议：标准数据条款为合同终止后 30 天数据导出窗口，到期后按安全标准删除，备份数据按技术上限处理，偏离标准须法务和数据安全负责人复核。Agent 使用要求：必须对标标准数据条款基准输出偏离项。

规则 LEG-042：价税分离条款审查。适用场景：合同税务条款审查、价税分离。触发条件：合同缺少价税分离条款。风险等级：高。处理建议：合同必须列明四要素即不含税价款、适用税率、税额和价税合计，须与财务确认税率适用正确、发票类型匹配和纳税义务时间合理。Agent 使用要求：必须提示缺少价税分离条款的税务合规风险。

规则 LEG-043：复合交易分项列明。适用场景：复合交易合同、混合销售。触发条件：合同同时涉及软件服务、货物销售和安装服务但未分项列明。风险等级：高。处理建议：软件服务、货物销售和安装服务须分项列明价款和适用税率，避免税务机关将混合销售从高适用税率，须与财务确认分项合理。Agent 使用要求：必须提示未分项列明的税务风险。

规则 LEG-044：发票合规担保条款。适用场景：发票条款审查、开票风险。触发条件：合同未约定发票真实性担保和换开责任。风险等级：中。处理建议：须约定供应商发票真实性担保条款，虚假发票须承担换开和赔偿责任，因开票不规范导致客户进项税转出须约定违约责任。Agent 使用要求：必须提示发票合规条款缺失风险。

规则 LEG-045：纳税义务时间匹配。适用场景：开票与付款节奏匹配、税务合规。触发条件：开票时间与付款节奏不匹配，存在提前开票垫税风险。风险等级：中。处理建议：开票时间须与付款节点匹配，避免在未收到款项时全额开票导致垫税，长期合同建议按付款节点分次开票。Agent 使用要求：必须提示开票与付款不匹配的现金流影响。

规则 LEG-046：跨境税务条款。适用场景：境外服务、跨境交易。触发条件：服务涉及境外客户但未约定代扣代缴增值税。风险等级：高。处理建议：境外服务须约定代扣代缴增值税和企业所得税的责任方、税率适用、完税凭证提供和税款承担方，须与财务和法务确认跨境税务合规路径。Agent 使用要求：不得在跨境税务路径未确认时输出条款可接受。

规则 LEG-047：渠道合同与直签差异。适用场景：渠道伙伴合同、渠道项目审查。触发条件：渠道合同未明确最终用户、服务责任主体和开票路径。风险等级：高。处理建议：渠道合同须明确最终用户身份、服务提供责任主体是供应商还是渠道商、开票对象和路径、服务范围和服务期与直签合同一致。Agent 使用要求：不得按直签合同标准判断渠道合同风险。

规则 LEG-048：渠道返点与折扣条款。适用场景：渠道返点结算、渠道合同审查。触发条件：渠道返点计算基准、支付条件和价格保护规则不明确。风险等级：中。处理建议：合同须明确返点计算基准是按合同金额还是实收金额、返点支付条件如客户回款后支付、价格保护规则和返点结算周期。Agent 使用要求：必须提示返点计算口径不清的回款风险。

规则 LEG-049：渠道冲突防护条款。适用场景：渠道冲突、客户归属争议。触发条件：渠道商之间或渠道与直销之间存在客户归属冲突。风险等级：高。处理建议：合同须明确客户归属规则、Deal Registration 保护期通常 180 天、保护期内其他渠道不得接触该客户和冲突仲裁机制。Agent 使用要求：不得在客户归属和冲突仲裁机制缺失时输出条款可接受。

规则 LEG-050：自动续费通知机制。适用场景：自动续费条款审查、续费流程。触发条件：合同包含自动续费但未明确通知机制。风险等级：中。处理建议：自动续费须提前 30 天书面通知客户，通知须包含续费价格、续费周期、取消窗口和确认机制，客户未在取消窗口内回复视为同意续费。Agent 使用要求：必须提示自动续费通知机制缺失的争议风险。

规则 LEG-051：续约价格调整条款。适用场景：续约涨价、合同续约。触发条件：续约涨价但未提前通知或合同无涨价条款。风险等级：中。处理建议：续约涨价须提前 60 天书面通知客户，通知须包含涨价原因、新价格和生效日期，客户可选择不续约但须在合同约定期限内通知。Agent 使用要求：不得在未确认涨价已通知时输出涨价可执行。

规则 LEG-052：续约宽限期规则。适用场景：合同到期续约、服务连续性。触发条件：合同到期后客户未及时续约但要求补办续约。风险等级：中。处理建议：到期后 30 天宽限期内补办续约可不中断服务，超出宽限期须按新合同重新评估价格和条款，宽限期内价格按原合同执行。Agent 使用要求：必须提示宽限期外的服务中断风险。

输出要求：合同风险 Agent 输出必须包含结论、风险等级、命中规则、条款风险点、修改建议、需人工确认角色和 CRM 写回草稿。结论取值包括风险较低、有条款风险、建议法务复核、暂不建议接受、信息不足。修改建议只能给谈判口径或需补充条款，不输出最终法律意见。

禁用边界：合同风险 Agent 不得输出“合同可以签”“法务已通过”“条款可接受”；不得在缺少合同正文、附件或客户模板时输出低风险；不得替代法务生成最终红线稿；不得承诺接受延期赔付、无限责任、源代码交付或数据安全绝对责任；不得忽略验收、付款、交付和财务之间的联动风险。

## 04 销售推进规则

以下内容来源于 04-销售推进规则，用于销售评估 Agent。

文档说明：本文档用于支撑销售评估 Agent 判断客户、商机、联系人、阶段、预算、竞品、跟进质量、下一步计划和审批说明。规则目标是帮助销售把商机推进动作结构化，而不是替代销售主管做成交判断。适用任务包括查询客户商机或联系人并做上下文绑定、总结商机进展和健康度、识别预算决策链竞品采购流程和跟进节奏风险、生成下一步跟进计划、生成审批说明草稿、生成 CRM 跟进记录草稿。不适用任务包括替代销售主管承诺成交概率、自动修改商机阶段或金额、自动写回客户承诺审批结论或合同结论、在证据不足时输出确定性赢单判断。

关键输入字段：销售评估 Agent 应优先读取客户名称、商机编号、商机阶段、商机金额、商机负责人、联系人和角色、预算信息、竞品信息、最近跟进记录、下一步计划、产品范围和审批上下文。若客户或商机对象不唯一，不得直接写回；若预算、决策人、竞品、采购流程缺失，不得输出高确定性成交判断。

风险等级定义：低风险表示信息完整且动作清晰，可给出常规跟进建议；中风险表示存在预算、联系人、竞品、跟进质量或阶段信息不足，建议补充信息或销售主管复核；高风险表示对象不唯一、阶段动作不匹配、单一联系人依赖、报价明显超预算、审批材料不足或写回存在越权风险；阻塞表示缺少唯一对象、缺少下一步关键字段或缺少审批必备材料，应先追问或要求人工确认。

规则 QRY-001：查询结果必须可消歧。适用场景：客户查询、商机查询、联系人查询。触发条件：查询命中 2 个及以上客户、商机或联系人。风险等级：中。处理建议：输出候选列表、匹配原因、负责人、阶段、金额、最近跟进和选择动作。Agent 使用要求：不得直接选择候选对象或写回。

规则 QRY-002：全局入口先绑定上下文。适用场景：全局对话入口、未指定对象的总结或评审。触发条件：用户从全局入口要求总结、评审或写回，但未指定客户或商机。风险等级：阻塞。处理建议：先要求选择客户或商机，不直接生成结论。Agent 使用要求：必须先完成对象绑定。

规则 QRY-003：查询结果权限控制。适用场景：字段权限、客户权限、商机权限。触发条件：用户无客户、商机或字段权限。风险等级：高。处理建议：仅展示允许字段，说明不可见原因和可操作动作。Agent 使用要求：不得泄露无权限字段。

规则 QRY-004：对象不唯一禁止写回。适用场景：CRM 写回前校验。触发条件：未绑定唯一 customer_id 或 opportunity_id。风险等级：阻塞。处理建议：禁用写回按钮，要求用户选择对象。Agent 使用要求：不得生成可保存状态。

规则 STG-001：阶段准入和退出条件。适用场景：商机总结、商机评审、下一步计划。触发条件：生成总结、评审或下一步计划时。风险等级：中。处理建议：检查当前阶段是否具备预算、关键人、需求、方案、采购路径等退出条件。Agent 使用要求：必须输出阶段健康度和缺失条件。

规则 STG-002：阶段与动作不匹配。适用场景：早期商机审批、后期商机缺信息。触发条件：早期商机申请高折扣或合同审批，后期仍缺预算或关键人。风险等级：高。处理建议：提示阶段动作不匹配，建议先补齐关键证据。Agent 使用要求：不得只按用户要求生成审批说明。

规则 STG-003：赢单概率与证据不一致。适用场景：赢率判断、商机健康度判断。触发条件：CRM 赢单概率高，但预算、决策人、竞品或下一步缺失。风险等级：中。处理建议：输出证据不足，不建议直接沿用高赢率。Agent 使用要求：不得输出高确定性成交判断。

规则 STG-004：长期停滞。适用场景：商机推进节奏判断。触发条件：多次跟进但阶段、关键人、预算、方案无变化。风险等级：中。处理建议：建议制定激活计划或重新评估商机质量。Agent 使用要求：必须提示停滞原因和下一步动作。

规则 CON-001：决策人缺失。适用场景：联系人角色评估。触发条件：当前商机缺少最终决策人、经济买方或采购负责人。风险等级：中。处理建议：建议补充决策链并安排关键人沟通。Agent 使用要求：不得假设当前联系人就是最终决策人。

规则 CON-002：角色覆盖不足。适用场景：决策链地图、客户关系分析。触发条件：缺少业务负责人、IT 或技术、采购、法务、财务、使用方任一关键角色。风险等级：中。处理建议：输出联系人角色地图和缺口。Agent 使用要求：必须区分已覆盖角色和缺失角色。

规则 CON-003：支持者与阻力人识别。适用场景：关系风险判断、下一步策略。触发条件：跟进记录出现支持、反对、质疑、拖延、转介绍等信号。风险等级：中。处理建议：标记 champion、blocker、concern 和下一步关系策略。Agent 使用要求：不得编造联系人态度。

规则 CON-004：单一联系人依赖。适用场景：单线推进风险。触发条件：仅依赖一个联系人推进，且该联系人不是决策人或采购负责人。风险等级：高。处理建议：建议拓展多角色联系人，避免单点风险。Agent 使用要求：必须提示关系覆盖风险。

规则 BUD-001：预算不明确。适用场景：预算判断、报价策略。触发条件：CRM 或用户输入缺少预算信息。风险等级：中。处理建议：要求确认预算范围、预算来源、预算 owner 和审批状态。Agent 使用要求：预算缺失时不得输出高确定性成交判断。

规则 BUD-002：预算成熟度分级。适用场景：预算状态判断。触发条件：预算为口头、模糊或未立项。风险等级：中。处理建议：输出预算状态，包括无预算、口头预算、预算范围、已立项、已审批、采购中。Agent 使用要求：必须说明预算成熟度。

规则 BUD-003：报价超预算。适用场景：报价与预算偏差。触发条件：报价高于预算上限，或预算低于方案最低可售范围。风险等级：高。处理建议：输出报价差距和可选方案，如降范围、分期、替代版本。Agent 使用要求：不得直接建议大幅折扣而不说明风险。

规则 BUD-004：采购路径缺失。适用场景：采购流程判断。触发条件：缺少招投标、采购审批、法务审查、财务付款流程。风险等级：中。处理建议：建议确认采购流程和关键时间点。Agent 使用要求：必须提示采购路径不清会影响成交周期。

规则 CMP-001：高折扣必须补竞品态势。适用场景：高折扣、客户压价、输赢判断、审批说明。触发条件：申请高折扣、客户压价、输赢判断或审批说明。风险等级：中。处理建议：要求补充竞品名称、报价差、能力差、客户倾向。Agent 使用要求：不得只写客户要求低价。

规则 CMP-002：在位厂商替换风险。适用场景：替换项目、竞品迁移。触发条件：客户已有在位系统或正在替换供应商。风险等级：中。处理建议：输出替换成本、迁移成本、关键差异和推进策略。Agent 使用要求：必须提示迁移和客户习惯风险。

规则 CMP-003：客户价值证明不足。适用场景：折扣理由、审批材料。触发条件：折扣理由仅为客户要求低价或竞品便宜。风险等级：中。处理建议：要求补充业务价值、ROI、标杆价值、扩展潜力。Agent 使用要求：不得把低价诉求当作充分审批理由。

规则 CMP-004：竞品报价不完整。适用场景：竞品报价对比。触发条件：用户提到竞品价格但缺少范围、服务期、模块、实施费或付款条件。风险等级：中。处理建议：不直接比较价格，先提示比较口径不足。Agent 使用要求：不得输出简单价格优劣判断。

规则 FOL-001：14 天无跟进。适用场景：跟进节奏判断。触发条件：最近一次跟进距当前超过 14 天。风险等级：中。处理建议：提示推进风险，建议尽快安排客户沟通。Agent 使用要求：必须提示跟进空窗期。

规则 FOL-002：跟进记录质量评分。适用场景：商机总结、跟进记录写回。触发条件：生成总结、评估进展或写回记录。风险等级：中。处理建议：检查是否包含客户反馈、异议、下一步、联系人、时间和证据。Agent 使用要求：不得把空泛记录作为强证据。

规则 FOL-003：空泛跟进不得作强证据。适用场景：跟进记录质量判断。触发条件：记录只有已沟通、继续推进、客户有意向等空泛表达。风险等级：中。处理建议：标记为弱证据，要求补充真实反馈和下一步。Agent 使用要求：必须提示证据不足。

规则 NXT-001：下一步计划可执行性。适用场景：生成下一步跟进计划。触发条件：用户要求生成跟进计划。风险等级：低。处理建议：必须包含 owner、对象、时间、方式、目标、成功标准和依赖项。Agent 使用要求：不得只输出泛泛建议。

规则 NXT-002：下一步缺关键字段。适用场景：下一步计划草稿。触发条件：缺少对象、时间、目标、owner 任一字段。风险等级：阻塞。处理建议：输出追问或草稿状态为 needs_input。Agent 使用要求：不得生成可直接保存的计划。

规则 NXT-003：客户承诺需谨慎。适用场景：客户话术、邮件、回复。触发条件：用户要求生成对客户话术、邮件或回复。风险等级：中。处理建议：避免承诺未审批折扣、未确认交付日期或未审核条款。Agent 使用要求：必须使用待内部确认的谨慎表达。

规则 APR-001：审批说明完整性。适用场景：折扣、付款、交付、合同例外审批说明。触发条件：生成折扣、付款、交付、合同例外审批说明。风险等级：中。处理建议：包含申请事项、政策例外、业务理由、客户价值、财务影响、风险缓释和替代方案。Agent 使用要求：不得生成缺少业务理由的审批说明。

规则 APR-002：审批路径自动路由。适用场景：审批角色判断。触发条件：折扣低于阈值、账期超标、验收后付款、交付压缩、合同赔付。风险等级：高。处理建议：输出需要参与的审批角色，包括销售主管、财务、交付、法务、管理层。Agent 使用要求：必须提示复核角色。

规则 APR-003：多约束触发复杂商机评审。适用场景：多视角协同判断。触发条件：客户同时提出折扣、付款、交付、合同中的两类及以上条件。风险等级：高。处理建议：建议进入多视角协同判断，不由单一 Agent 给总体结论。Agent 使用要求：必须触发销售、财务、交付、合同等视角的协同判断。

规则 APR-004：审批材料不足。适用场景：提交审批前检查。触发条件：缺少客户价值、竞品、毛利或回款影响、交付范围、合同条款。风险等级：阻塞。处理建议：先补充材料，再生成正式审批说明草稿。Agent 使用要求：不得输出完整审批说明。

规则 WB-001：写回字段白名单。适用场景：CRM 写回。触发条件：用户点击保存、创建计划、更新商机字段。风险等级：中。处理建议：只允许写回跟进记录、跟进计划、AI 评审摘要、待确认事项等白名单字段。Agent 使用要求：不得写回审批通过或客户承诺。

规则 WB-002：写回差异预览。适用场景：覆盖已有字段。触发条件：写回字段已有值，或用户编辑过 AI 草稿。风险等级：中。处理建议：展示当前值、新值、差异摘要和覆盖确认。Agent 使用要求：未经确认不得覆盖。

规则 WB-003：可编辑草稿版本。适用场景：编辑、重新生成、保存失败重试。触发条件：用户编辑、重新生成或保存失败后重试。风险等级：低。处理建议：保留 AI 原文、用户改动和版本号。Agent 使用要求：用户改动优先于 AI 原文。

规则 WB-004：写回幂等与重试。适用场景：写回超时、接口失败、用户重试。触发条件：写回超时、接口失败或用户点击重试。风险等级：中。处理建议：使用幂等标识，避免重复创建跟进记录。Agent 使用要求：必须保留草稿并提示重试。

规则 WB-005：禁止写回审批结论。适用场景：写回内容校验。触发条件：草稿包含已批准、已承诺、合同可签等内容。风险等级：高。处理建议：阻止写回或要求人工修改。Agent 使用要求：必须拦截越权结论。

规则 CHN-001：渠道伙伴分级。适用场景：渠道伙伴管理、渠道商机评估。触发条件：渠道商机缺少伙伴等级标识或等级与折扣不匹配。风险等级：中。处理建议：渠道伙伴按 ARR 门槛和能力认证分为 Silver、Gold、Platinum 三级，Silver 门槛为年 ARR 50 万以上，Gold 为 200 万以上，Platinum 为 500 万以上，每级须通过对应技术认证和销售认证。Agent 使用要求：不得在伙伴等级未确认时输出渠道折扣建议。

规则 CHN-002：渠道折扣基线。适用场景：渠道折扣审批、渠道商机报价。触发条件：渠道折扣超出对应等级的折扣基线。风险等级：高。处理建议：Silver 伙伴基础折扣为 20%，Gold 为 30%，Platinum 为 35%，超出基线须额外审批并说明战略理由，渠道折扣不得低于产品底价。Agent 使用要求：必须核对伙伴等级与折扣基线匹配。

规则 CHN-003：Deal Registration 保护。适用场景：渠道商机报备、渠道冲突防护。触发条件：渠道商机缺少 Deal Registration 或保护期已过期。风险等级：高。处理建议：首注伙伴享额外 5-10% 折扣保护，Deal Registration 保护期为 180 天，保护期内其他渠道不得接触该客户，过期后须重新报备。Agent 使用要求：不得在 Deal Registration 缺失或过期时输出折扣保护建议。

规则 CHN-004：渠道佣金模型选择。适用场景：渠道佣金结构设计、佣金方案审批。触发条件：渠道佣金模型未明确或模型选择不合理。风险等级：中。处理建议：根据伙伴类型选择佣金模型，统一费率适合简单产品线，阶梯佣金适合成长型伙伴，收入分成适合深度合作，混合模型适合多产品线。Agent 使用要求：不得在佣金模型未确认时输出佣金计算结果。

规则 CHN-005：渠道伙伴利润底线。适用场景：渠道利润评估、渠道报价审批。触发条件：渠道伙伴利润低于行业底线。风险等级：高。处理建议：VAR 类型伙伴毛利底线为 25-35%，MSP 类型为 30-40%，分销商类型为 8-15%，低于底线须评估伙伴合作意愿和替代方案。Agent 使用要求：必须输出渠道伙伴利润空间评估。

规则 CHN-006：MDF 市场发展基金。适用场景：渠道市场活动、MDF 审批。触发条件：渠道伙伴申请 MDF 但未按规则使用或超出比例。风险等级：中。处理建议：MDF 通常为伙伴收入的 2-5%，按季度审批拨付，须提供活动计划、预算和效果评估，MDF 不得用于日常运营或渠道返点替代。Agent 使用要求：不得在 MDF 使用计划未审核时输出可拨付建议。

规则 CHN-007：渠道冲突与客户归属。适用场景：渠道冲突、客户归属争议。触发条件：直销与渠道对同一客户存在归属冲突或渠道间交叉报备。风险等级：高。处理建议：直销与渠道客户归属须明确规则，已有 Deal Registration 的客户归属报备伙伴，模糊地带优先协商解决，冲突仲裁由渠道管理团队裁定。Agent 使用要求：必须提示客户归属冲突风险。

规则 CHN-008：渠道单审批路径。适用场景：渠道商机折扣审批、渠道合同审批。触发条件：渠道商机折扣审批缺少渠道经理会签。风险等级：中。处理建议：渠道单折扣审批流程与直签一致，额外须渠道经理会签确认伙伴等级、Deal Registration 有效性和返点计算口径。Agent 使用要求：不得在渠道经理未会签时输出审批通过建议。

规则 RFP-001：招投标场景识别。适用场景：客户发起 RFP 或招标、采购流程判断。触发条件：客户发起 RFP、招标或询价流程。风险等级：中。处理建议：须在 CRM 中标记采购路径为招投标，记录招标方、招标编号、截止时间和评标标准，招投标商机须按招投标流程管理。Agent 使用要求：不得将招投标商机按普通商机处理。

规则 RFP-002：投标资质前置检查。适用场景：投标前资质评估。触发条件：客户发起招标但未确认公司资质是否满足要求。风险等级：高。处理建议：须确认公司营业执照、行业资质、案例要求、财务报告、ISO 认证、安全资质和信用评级是否满足招标文件要求，任一不满足须评估是否具备投标资格。Agent 使用要求：不得在资质检查未完成时输出可投标建议。

规则 RFP-003：投标保证金评估。适用场景：投标保证金、资金占用评估。触发条件：招标要求缴纳投标保证金。风险等级：中。处理建议：须评估保证金金额、退还条件和资金占用风险，保证金通常为合同金额的 2-5%，退还周期为中标后签约或未中标后 30 天内。Agent 使用要求：必须提示保证金资金占用风险。

规则 RFP-004：中标后合同变更限制。适用场景：中标后合同谈判、合同条款审查。触发条件：中标后合同条款偏离招标文件要求。风险等级：高。处理建议：中标后合同条款不得实质性偏离招标文件要求，包括价格、服务范围、交付周期和付款条件，偏离可能导致中标无效或违约。Agent 使用要求：必须提示中标后合同变更的法律风险。

规则 RFP-005：框架协议执行规则。适用场景：框架协议下单、额度管理。触发条件：客户在框架协议下下单但未确认额度、期限和交付范围。风险等级：中。处理建议：框架协议下单须确认剩余额度、协议期限、交付范围是否在框架内，超框架范围须签署补充协议。Agent 使用要求：不得在框架额度未确认时输出可执行建议。

规则 RFP-006：投标报价策略。适用场景：投标报价、竞品分析。触发条件：投标报价未评估竞品报价、客户预算和成本底线。风险等级：高。处理建议：投标报价须评估竞品报价水平、客户预算上限、成本底线和利润率目标，报价过高降低中标概率，报价过低影响利润和交付质量。Agent 使用要求：不得在未评估竞品和成本时输出报价建议。

规则 KPI-001：审批周期 SLA。适用场景：Deal Desk 审批效率监控。触发条件：审批请求超过 SLA 时限。风险等级：中。处理建议：常规审批须在 24 小时内完成，复杂审批须在 48 小时内完成，超过 72 小时须增配资源或升级处理。Agent 使用要求：审批超时须提示加速处理。

规则 KPI-002：折扣泄漏率监控。适用场景：折扣管理、定价效率评估。触发条件：折扣泄漏率超过目标值。风险等级：高。处理建议：折扣泄漏率为折扣总额除以目录总价，目标值低于 8%，超过 12% 须排查折扣审批流程漏洞和折扣授权线合理性。Agent 使用要求：必须输出折扣泄漏率计算和趋势分析。

规则 KPI-003：Desk 捕获率。适用场景：Deal Desk 覆盖率监控。触发条件：超阈值交易未经过 Deal Desk 审批。风险等级：高。处理建议：超阈值交易经 Deal Desk 审批的比例须高于 95%，低于 95% 须排查是否存在绕过审批流程的情况。Agent 使用要求：必须提示 Desk 捕获率不足的风险。

规则 KPI-004：报价响应时间。适用场景：报价效率、客户响应速度。触发条件：从报价请求到报价发出的时间超出目标值。风险等级：中。处理建议：报价响应时间影响客户体验和竞争力，标准报价须在 4 小时内响应，复杂报价须在 24 小时内响应。Agent 使用要求：必须提示报价响应延迟对竞争力的影响。

规则 KPI-005：审批记录完整性。适用场景：审批流程审计、交易记录管理。触发条件：交易记录缺少非标条款或折扣原因记录。风险等级：中。处理建议：每个非标条款和折扣原因须完整记录在交易记录中，包括申请理由、审批角色、审批结果和风险缓释措施。Agent 使用要求：不得在审批记录不完整时输出审批已完成。

规则 KPI-006：成交后移交流程。适用场景：商机成交后、客户成功移交。触发条件：商机 Closed-Won 后财务、CS 和续约数据未自动移交。风险等级：中。处理建议：商机 Closed-Won 后须自动将财务数据移交财务团队、客户信息移交 CS 团队、续约数据移交续约团队，确保数据连续性和客户体验。Agent 使用要求：必须提示成交后移交的完整性要求。

规则 REN-001：续费启动时间。适用场景：续费商机创建、续费流程启动。触发条件：合同到期前未在规定时间创建续费交易。风险等级：中。处理建议：标准客户合同到期前 90 天自动创建续费交易，企业级客户 180 天，续费交易须关联原合同和原产品线。Agent 使用要求：必须提示续费启动延迟的风险。

规则 REN-002：T-minus 续费流程。适用场景：续费流程管理、续费时间线。触发条件：续费交易未按 T-minus 流程推进。风险等级：中。处理建议：T-minus 续费流程为 T-90 创建续费交易、T-60 进行价值回顾、T-30 提交续费提案、T-14 启动谈判、T-0 完成续约关闭。Agent 使用要求：必须输出续费时间线和当前阶段。

规则 REN-003：续费增购捆绑。适用场景：续费与增购联动、续费价值最大化。触发条件：续费时未评估增购机会。风险等级：中。处理建议：约 68% 的增购发生在续费时刻，续费流程须纳入增购评估，包括席位扩展、版本升级、新模块交叉销售和多产品线拓展。Agent 使用要求：不得在续费时忽略增购评估。

规则 REN-004：续费健康度指标。适用场景：续费健康度评估、续费风险监控。触发条件：NRR 或 GRR 低于目标值。风险等级：高。处理建议：NRR 净收入留存率目标为 110% 以上，GRR 总收入留存率目标为 95% 以上，低于目标须排查流失原因、降级原因和增购不足原因。Agent 使用要求：必须输出 NRR 和 GRR 计算及趋势。

规则 REN-005：续费风险信号。适用场景：续费风险预警、续费流失预防。触发条件：合同到期前 30 天仍未启动续费流程。风险等级：高。处理建议：到期前 30 天未启动续费流程标记为高风险，须排查客户使用活跃度、客户满意度和竞品替换风险，制定紧急续费方案。Agent 使用要求：必须提示续费流失风险并输出紧急方案。

禁用边界：销售评估 Agent 不得在预算、决策人、竞品、采购流程缺失时输出高确定性成交判断；不得假设当前联系人就是最终决策人；不得自动修改商机阶段、赢率、金额或合同状态；不得生成对客户的确定性承诺，如已批准折扣、保证上线、合同条款没问题；不得将空泛跟进记录作为强证据。

以下内容用于约束协调 Agent 的回答质量、责任边界和 CRM 写回业务校验。

文档说明：本章节只保留可被业务问题检索使用的回答规范、人工确认边界、写回规则和失败兜底。公开响应 JSON、前端卡片字段、按钮状态和节点运行状态属于接口与实现协议，应在 Chatflow 和接口文档中维护，不作为业务知识召回内容。

输出设计原则：AI Deal Desk 应先给直接结论，再围绕最重要的业务问题组织证据和动作。相同事实、风险、建议和结论只表达一次；CRM 写回内容必须是草稿并经过用户确认；部分失败时仍输出可用结果并说明影响范围；不得输出审批通过、合同可签、财务已通过、交付已承诺等越权结论。

模板 OUT-001：主题驱动的商机评审结论。适用场景：复杂商机评审，通常同时涉及销售、财务、交付、合同中的两个及以上维度。开头用 1-2 句话给出总体判断；随后默认围绕最重要的 3 个业务问题组织“判断、必要证据、业务影响、对应动作”，只有确有互不重复且会显著改变决策的重大问题时才扩展到 5 个；最后汇总 3-5 个可执行下一步和会实质改变结论的待确认信息。不得按四个 Agent 机械生成四个完整章节，不得在结尾重复开头结论。用户明确要求四个视角时，优先使用紧凑矩阵呈现各视角的独有判断。默认不包含 CRM 写回草稿；只有用户明确要求保存或写入时，才进入写回预览与人工确认。

模板 OUT-002：财务风控判断。适用场景：用户需要判断折扣、付款、账期或回款条件。回答应包含直接结论、支持结论的财务事实或规则、关键影响、建议条件和必要的人工复核；不得输出折扣已批准、财务已通过或可以直接承诺账期。规则等级仅作为证据，不自动等同于整体商机风险。

模板 OUT-003：交付可行性判断。适用场景：用户需要判断上线周期、范围、资源、集成或验收准备。回答应包含直接结论、必要排期或范围假设、关键影响、客户前置条件和建议动作；不得输出保证上线日期、交付资源已锁定或定制需求已确认包含在标准交付中。

模板 OUT-004：合同风险判断。适用场景：用户需要判断验收、赔付、责任边界、数据安全或特殊承诺。回答应包含直接结论、相关条款事实或规则、关键影响、修改建议和必要的法务复核；不得输出合同可以签、法务已通过或条款可直接接受。

模板 OUT-005：商机总结。适用场景：用户要求总结客户、商机或近期进展。只总结用户要求的范围；除非用户同时要求评审、建议或写回，不自动追加商机健康度、风险、下一步建议或可写回草稿。相同 CRM 事实只出现一次，不得编造 CRM 中不存在的跟进记录。

CRM 写回白名单规则：允许在用户明确要求并确认后写回跟进记录、跟进计划、AI 评审摘要和审批说明草稿。跟进记录可包含客户诉求、沟通摘要、AI 初步风险、待确认事项和下一步动作；跟进计划可包含 owner、对象、时间、方式、目标和成功标准；AI 评审摘要可包含评审结论、风险等级、命中规则和需复核角色；审批说明草稿可包含申请事项、业务理由、风险缓释、替代方案和所需材料。Agent 使用要求：只写入白名单字段，且不得把普通分析默认转成写回草稿。

CRM 写回禁止规则：禁止自动写回审批通过结论、正式报价承诺、合同可签结论、交付日期承诺、财务授信结论、客户未经确认的口头承诺、覆盖已有关键字段但未展示差异的内容。Agent 使用要求：若草稿包含已批准、已承诺、合同可签等越权表达，必须阻止写回或要求人工修改。

写回前校验规则：写回前必须校验对象唯一性、字段白名单、用户确认、差异预览、禁用词检查、幂等处理和权限检查。对象唯一性要求绑定唯一 customer_id 或 opportunity_id；字段白名单要求只能写入允许字段；用户确认要求必须点击确认，不能自动保存；差异预览要求覆盖已有内容前展示差异；禁用词检查要求不得包含已批准、已承诺、合同可签等越权结论；幂等处理要求重试时不得重复创建记录；权限检查要求当前用户必须有目标对象写权限。

失败兜底规则 CRM 查询失败：当 CRM 查询失败时，应说明查询失败，保留用户输入分析，提示重试或手动选择对象。Agent 使用要求：不得编造 CRM 数据。

失败兜底规则 知识库召回为空：当知识库召回为空时，应输出基于通用规则的初步判断，并标记需人工复核。Agent 使用要求：不得伪造命中规则编号。

失败兜底规则 专家 Agent 部分失败：当某个专家 Agent 失败时，应输出其他 Agent 结果，标记 partial_success 和失败影响。Agent 使用要求：必须说明缺失哪个专家结论以及对总评估的影响。

失败兜底规则 对象不唯一：当客户或商机对象不唯一时，应输出候选列表，不做写回。Agent 使用要求：必须等待用户选择。

失败兜底规则 缺少关键字段：当缺少关键字段时，应输出缺失字段和追问，不输出低风险结论。Agent 使用要求：可继续输出非阻塞建议，但必须标记不确定性。

失败兜底规则 写回失败：当写回失败时，应保留草稿，提示重试，不重复创建记录。Agent 使用要求：必须使用幂等标识避免重复写回。

禁用边界：所有输出不得展示模型原始思维链；不得把 Agent 建议包装成审批结论；不得在缺少关键字段时输出低风险；不得用自然语言隐藏越权承诺；不得自动写回未经用户确认的内容；不得泄露用户无权限查看的客户、合同或财务字段。

## 06 CordysCRM 对象与 L2C 链路

以下内容来源于 CordysCRM skills 的底座对象与业务链路整理，用于帮助 AI Deal Desk 在回答客户、商机、合同、回款和发票相关问题时保持 CRM 业务语义一致。AI Deal Desk 的主角仍然是商机协同工作台，不重做 CordysCRM 底座能力；这些规则只用于让 Agent 正确理解对象关系、读取范围和写回边界。

规则 CORDYS-L2C-001：CordysCRM 的主业务链路是 Lead-to-Cash。典型顺序为线索 Lead、客户 Account、联系人 Contact、商机 Opportunity、报价单 Quotation、合同 Contract、订单 Order、回款计划 PaymentPlan、回款记录 PaymentRecord、发票 Invoice。Agent 使用要求：分析商机时优先以客户和商机为主对象，报价、合同、回款和发票作为上下游证据，不把全部链路都塞进每次回答。

规则 CORDYS-L2C-002：客户和商机是 AI Deal Desk P0 的核心绑定对象。用户围绕“这家公司”“这个商机”“当前商机”提问时，Chatflow 应先确认是否已绑定唯一客户或唯一商机。Agent 使用要求：对象不唯一时先返回候选对象，不做评审和写回。

规则 CORDYS-L2C-003：线索到客户通常通过转化动作形成，不应假设每条线索天然都有客户 ID。Agent 使用要求：如果用户只提供线索或模糊客户名，应先确认目标客户或商机，再做后续分析。

规则 CORDYS-L2C-004：商机通过 customerId 关联客户。Agent 使用要求：商机评审必须能回到所属客户，读取客户基本信息、行业、负责人、历史商机、历史合同、近期跟进和回款概况；缺少客户信息时要说明判断不完整。

规则 CORDYS-L2C-005：报价单通过 opportunityId 关联商机。Agent 使用要求：涉及折扣、报价方案、成交策略、审批说明时，应把报价信息作为关键证据；如果报价口径、服务期、实施费或付款条件缺失，不得直接判断价格优劣。

规则 CORDYS-L2C-006：合同通常通过 customerId 关联客户，部分场景未必直接关联 opportunityId。Agent 使用要求：合同风险、验收条款、赔付责任、续费到期和历史履约判断，可以从客户侧回查合同；不得在没有证据时假设某合同一定属于当前商机。

规则 CORDYS-L2C-007：合同到现金链路包括合同、回款计划、回款记录和发票。Agent 使用要求：财务 Agent 判断付款风险时，不能只看用户口头输入，还应关注是否已有回款计划、计划是否到期、实际回款是否匹配、发票是否已开且是否回款。

规则 CORDYS-L2C-008：客户 360 视角应聚合客户基本信息、名下商机、名下合同、回款概况、发票概况、关键联系人和最近跟进。Agent 使用要求：当用户问“看看这家公司最近怎么样”时，输出应围绕业务判断组织，不展示底座对象关系图。

规则 CORDYS-L2C-009：商机分析视角应读取商机名称、客户、金额、阶段、负责人、最近跟进、下一步计划、报价折扣、付款条件、交付范围、上线时间、定制需求、合同条款和验收条件。Agent 使用要求：只读取本次任务必要信息，轻量问答不强制读取完整 L2C 链路。

规则 CORDYS-L2C-010：合同到现金链路常见风险包括合同已签但无回款计划、回款计划到期但无回款记录、已开票但未回款、已回款但未开票、合同金额与计划金额或实收金额存在明显缺口。Agent 使用要求：命中这些信号时，应提示财务或商务复核，不输出审批通过或客户承诺。

规则 CORDYS-L2C-011：文件上传只作为本轮补充材料，不改变客户或商机绑定关系。Agent 使用要求：用户上传会议纪要、合同片段或报价材料时，应把文件内容作为辅助证据；如果文件内容与 CRM 绑定对象冲突，应提示用户确认，不自动覆盖 CRM 数据。

规则 CORDYS-L2C-012：CRM 写回前必须确认唯一客户或商机。Agent 使用要求：未绑定对象、对象候选多于一个、目标对象权限不明确时，不生成可保存状态，只生成待确认草稿或对象选择卡。

## 07 CordysCRM 风险信号参考

以下内容用于帮助多 Agent 从 CordysCRM 业务对象中识别异常信号。风险信号只用于提醒人工复核和生成下一步建议，不代表审批结论，不替代销售、财务、交付、商务或法务的最终判断。

规则 CORDYS-RISK-001：风险输出优先呈现最关键的 1 到 3 条，不堆砌所有异常。Agent 使用要求：先说明异常对成交、回款、交付或合同责任的影响，再给下一步动作。

规则 CORDYS-RISK-002：线索超过 3 天未跟进，表示线索热度可能下降。建议动作：提醒销售今日优先联系，确认客户当前需求和下一步沟通时间。

规则 CORDYS-RISK-003：商机在当前阶段停留超过 7 天，表示推进可能卡住。建议动作：确认卡点、决策人、预算、采购路径和下一步动作。

规则 CORDYS-RISK-004：今日跟进计划未完成，表示销售执行动作存在延误。建议动作：整理待办优先级，优先处理高金额、高阶段或临近关键节点的商机。

规则 CORDYS-RISK-005：客户超过 90 天无跟进，表示客户活跃度下降。建议动作：安排复联，重新评估客户价值、需求变化和是否仍有推进机会。

规则 CORDYS-RISK-006：客户有活跃商机但超过 60 天无跟进，表示商机可能失控。建议动作：补充跟进记录，确认当前推进计划，必要时重新评估商机阶段和赢单概率。

规则 CORDYS-RISK-007：回款逾期表示现金回收风险增加。建议动作：由销售确认客户付款安排，财务跟进催收；Agent 不得输出“客户会付款”这类确定性结论。

规则 CORDYS-RISK-008：未来 7 天有多笔回款到期，表示短期回款压力集中。建议动作：提前确认付款节点、开票状态、客户内部审批状态和责任人。

规则 CORDYS-RISK-009：合同已签但无回款计划，表示合同到现金链路断裂。建议动作：补建回款计划或确认合同付款条款，必要时由财务复核。

规则 CORDYS-RISK-010：合同签约超过 30 天无回款记录，表示履约后回款风险。建议动作：核查客户付款状态、付款条件和是否存在验收或开票前置条件。

规则 CORDYS-RISK-011：回款计划已到期但无回款记录，表示计划执行偏差。建议动作：标记逾期并推动催收，输出应说明金额、到期日和责任人。

规则 CORDYS-RISK-012：已开发票但未回款，表示开票后应收风险。建议动作：核查客户付款进度、账期和逾期责任；需要时由财务复核。

规则 CORDYS-RISK-013：已回款但未开发票，表示发票流程缺口。建议动作：提醒商务或财务补充开票流程，不把它误判为成交风险本身。

规则 CORDYS-RISK-014：合同审批超过 7 天未完成，表示合同流程卡点。建议动作：追踪当前审批节点、卡点原因和需要补充的材料。

规则 CORDYS-RISK-015：合同被同一节点驳回多次，表示条款或资料存在反复问题。建议动作：汇总驳回原因，先补齐资料或调整条款后再提交。

规则 CORDYS-RISK-016：合同到期小于等于 30 天且未续约，表示续费风险。建议动作：提醒销售确认续约意向、预算、采购流程和竞品替换可能。

规则 CORDYS-RISK-017：签约合同缺附件，表示合同归档不完整。建议动作：补齐附件、审批记录或相关补充协议。

规则 CORDYS-RISK-018：合同存在特殊条款，尤其是验收、赔付、延期责任、数据安全、付款前置条件或客户单方解除条款，表示商务或法务责任边界不清。建议动作：建议商务或法务复核，不输出“合同可签”。

规则 CORDYS-RISK-019：商机赢单但未找到关联合同，表示赢单后未签约、漏建合同或链路断裂。建议动作：检查商机阶段是否准确，确认合同创建和签约计划。

规则 CORDYS-RISK-020：合同签约后长期未开票，表示发票流程滞后。建议动作：核查开票条件是否满足，以及是否影响客户付款。

规则 CORDYS-RISK-021：接口或工具查询失败时，不得把数据缺失解释为“没有风险”。Agent 使用要求：区分“确实没有数据”和“数据读取失败”，后者应输出需重试或人工复核。

规则 CORDYS-RISK-022：风险信号进入页面时，应转换为用户语言。可写为"当前方案回款风险偏高，原因是尾款依赖验收后支付，建议先确认验收标准和回款节点"，不要展示原始接口响应、内部 prompt 或调试字段。

规则 CORDYS-RISK-023：渠道单缺少 Deal Registration 或伙伴等级不匹配，表示渠道合规风险。建议动作：确认 Deal Registration 是否有效、伙伴等级与折扣是否匹配、是否需渠道经理复核。

规则 CORDYS-RISK-024：合同到期时间小于 60 天且无续费交易创建，表示续费流失高风险。建议动作：立即创建续费交易，确认客户续费意向和竞品动态，输出续费紧急方案。

规则 CORDYS-RISK-025：合同缺少价税分离条款或发票类型未确认，表示税务合规风险。建议动作：联动财务确认税率适用和发票类型，补签价税分离条款。

规则 CORDYS-RISK-026：实施费低于订阅年费 15% 且含定制开发，表示实施成本覆盖不足。建议动作：评估实施实际成本和交付范围，确认是否需要调整实施费或缩减范围。

规则 CORDYS-RISK-027：大额商机缺少客户分级或战略价值评估，表示分级管理缺失。建议动作：补全客户分级信息，按分级标准评估折扣授权线和战略价值。

规则 CORDYS-RISK-028：多年合同锁价期即将到期但未启动续约谈判，表示续费窗口风险。建议动作：提前启动续约谈判，确认新周期价格策略和客户续约意向。

## 08 CordysCRM 角色视角参考

以下内容用于约束 AI Deal Desk 多 Agent 的分工、Prompt 语气和汇总方式。P0 页面面向销售工作台，但复杂商机评审需要模拟销售、财务、交付、合同等不同视角，再汇总成轻量结论、风险和下一步建议。

规则 CORDYS-ROLE-001：销售视角关注“我接下来该做什么”。输出应行动导向，具体到联系谁、确认什么、什么时候做、推进目标是什么。Agent 使用要求：先给下一步，再解释原因。

规则 CORDYS-ROLE-002：销售 Agent 应回答当前商机推进状态、主要卡点、下一步应确认的信息、是否需要拉财务/交付/商务/法务复核，以及是否适合生成跟进记录或计划草稿。Agent 使用要求：只有用户明确要求保存或写入时，才进入写回确认。

规则 CORDYS-ROLE-003：销售经理视角关注“谁需要我关注”。输出可用于风险升级、团队卡点和重点商机提示。Agent 使用要求：在 P0 中不要把销售经理作为常驻角色展示，只在需要人工复核或升级时表达为“建议销售主管复核”。

规则 CORDYS-ROLE-004：财务视角关注“钱在哪里”。重点包括合同金额、付款条件、账期、首付款、尾款、回款计划、实际回款、发票和逾期。Agent 使用要求：金额、日期和节点优先；不得输出“财务已通过”或“账期可以承诺”。

规则 CORDYS-ROLE-005：财务 Agent 应回答当前付款方案是否增加回款风险，首付、尾款、账期和验收后付款是否存在问题，是否需要提高首付或缩短账期，是否缺少回款计划、发票或合同数据，以及哪些结论需要财务人工确认。

规则 CORDYS-ROLE-006：商务/合同视角关注“合同签对了吗”。重点包括合同审批状态、条款完整性、验收口径、赔付责任、责任边界、到期续约、附件归档和签约后开票回款衔接。Agent 使用要求：只能给内部风险识别和复核建议，不给最终法律意见。

规则 CORDYS-ROLE-007：合同 Agent 应回答哪些条款可能带来合同或履约风险，验收口径是否清晰，延期赔付和责任边界是否需要法务复核，是否存在审批或附件完整性问题，以及哪些事项应交由商务或法务确认。

规则 CORDYS-ROLE-008：交付视角关注“能不能按这个范围和时间交付”。重点包括上线周期、定制范围、数据迁移、接口集成、客户资源、验收标准和分阶段交付。Agent 使用要求：不承诺资源和上线日期，只给范围冻结、阶段拆分和验收前置建议。

规则 CORDYS-ROLE-009：交付 Agent 应优先使用 AI Deal Desk 已有交付规则，再结合 CordysCRM 中的商机、合同、订单和跟进记录补充判断。CordysCRM skill 对交付规则覆盖较弱，不应替代本项目交付知识库。

规则 CORDYS-ROLE-010：协调总结 Agent 按业务主题组织商机评审，不按销售、财务、交付、合同逐章复述。输出先给 1-2 句总体判断，默认保留最重要的 3 个业务问题，确有互不重复的重大问题时最多 5 个，再给 3-5 个可执行下一步和会实质改变结论的待确认信息。Agent 使用要求：相同事实、风险、动作和结论只表达一次；开头已给出的总体判断不得在结尾再次总结。

规则 CORDYS-ROLE-011：页面输出可以说“从财务角度看”“从合同角度看”，但不要展示 finance_agent、legal_agent、full_review、route_label 等内部标识。Agent 使用要求：前端画面只出现真实销售用户能理解的业务语言。

规则 CORDYS-ROLE-012：多 Agent 汇总时，AI 的职责是生成判断和草稿，不是直接替人审批。对分析、判断或评审类回答，全文只保留一次统一责任边界：AI 输出是内部评审建议，折扣、付款、交付承诺和合同条款需相应负责人确认。CRM 写回仍需单独经过用户确认。

规则 CORDYS-ROLE-013：渠道经理视角关注"渠道合规了吗"。重点包括伙伴等级是否匹配、Deal Registration 是否有效、折扣是否在等级基线内、返点结算口径是否正确、渠道冲突是否存在。Agent 使用要求：渠道经理视角须输出伙伴等级核对结果和折扣合规性判断，不得在渠道经理未确认时输出渠道折扣审批通过。

规则 CORDYS-ROLE-014：续费经理视角关注"客户会续费吗"。重点包括续费时间线是否按 T-minus 推进、NRR 和 GRR 是否达标、增购机会是否已评估、流失风险信号是否存在、合同到期时间是否紧迫。Agent 使用要求：续费经理视角须输出续费健康度评估和流失风险预警，不得在续费流程未启动时输出续费风险低。

## 09 知识库使用边界

本章节只说明 Dify 知识库的使用范围，不作为 Agent Prompt 或运行约束。Agent 行为约束、内部标签隐藏、轻量评审、不默认写回、对象绑定优先级和人工确认规则，已经迁移到 Chatflow 节点 Prompt 与本地文档 `docs/12-Chatflow Prompt 强约束清单.md`。

知识库使用原则：本页面用于提供业务事实、规则依据、对象关系和风险信号。Dify 检索到本页面内容时，Agent 只应引用与当前问题直接相关的业务规则，不应把知识库章节当成系统提示词执行。

入库建议：当前阶段继续使用统一单页知识库。源文档可以在本地按财务、交付、合同、销售、底座对象和回答质量拆分维护；Dify 侧仍优先绑定本页面。若未来多知识库绑定稳定，再按角色或业务域拆分。
