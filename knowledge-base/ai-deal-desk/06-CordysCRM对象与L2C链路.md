# CordysCRM 对象与 L2C 链路

## 用途

本文档用于给 多Agent智能助手 的 Chatflow、Agent Prompt 和外接知识库提供 CordysCRM 底座对象关系。它来自 `docs/reference/cordyscrm-skills/` 中的 CordysCRM skill 原文资料，并按 多Agent智能助手 的商机协同场景重新整理。

多Agent智能助手 不重做 CordysCRM 的 CRM 底座能力，但需要理解客户、商机、合同、回款、发票等对象如何关联，才能在回答商机问题、生成判断过程和整理写回内容时不脱离真实业务链路。

## L2C 主链路

CordysCRM 的核心业务链路是 Lead-to-Cash：

```text
线索 Lead
→ 客户 Account
→ 联系人 Contact
→ 商机 Opportunity
→ 报价单 Quotation
→ 合同 Contract
→ 订单 Order
→ 回款计划 PaymentPlan
→ 回款记录 PaymentRecord
→ 发票 Invoice
```

对 多Agent智能助手 来说，最重要的主对象是客户和商机；合同、报价、回款计划、回款记录和发票是分析商机风险时的上下游依据。

## 已知关联关系

| 源对象 | 目标对象 | 关联方式 | 对 多Agent智能助手 的意义 |
| --- | --- | --- | --- |
| 线索 | 客户 | 通过转化 API，不是简单字段关联 | 不能假设线索一定有客户 ID |
| 商机 | 客户 | `customerId` | 商机分析必须能回到客户 |
| 报价单 | 商机 | `opportunityId` | 折扣、报价和成交方案可从报价链路补充 |
| 合同 | 客户 | `customerId` | 合同没有直接 `opportunityId` 时，需通过客户侧关联判断 |
| 订单 | 合同 | `contractId` | 成交后交付与履约可从订单链路补充 |
| 回款计划 | 合同 | `contractId` | 财务风险分析要看计划回款节点 |
| 回款记录 | 回款计划 | `paymentPlanId` | 判断是否按计划回款 |
| 回款记录 | 合同 | `contractId` | 可以按合同汇总实际回款 |
| 发票 | 合同 | `contractId` | 判断开票与回款是否匹配 |

## Customer 360 视角

当用户问“看看某家公司”“这个客户最近怎么样”时，不应只看客户基本信息。应围绕客户聚合以下信息：

- 客户基本信息：名称、行业、地区、负责人。
- 名下商机：商机数量、金额、阶段、负责人、最近跟进。
- 名下合同：合同数量、签约金额、合同状态、到期时间。
- 回款概览：计划回款、实际回款、逾期情况。
- 发票概览：开票金额、未开票合同、已开票未回款。
- 联系人：关键联系人及其角色。
- 最近跟进：最近几条跟进记录或计划。

多Agent智能助手 P0 不需要把这些信息全部常驻在页面上；只有在用户提问或 Agent 分析需要时，作为回答依据或判断过程中的“已读取客户资料”展示。

## 商机分析视角

当用户围绕商机提问时，应优先确认唯一商机，然后读取：

- 商机名称、所属客户、金额、阶段、负责人。
- 最近跟进记录和下一步计划。
- 报价和折扣信息。
- 付款条件和账期。
- 交付范围、上线时间、定制需求。
- 合同条款、验收条件、赔付责任。
- 关联合同、回款、发票情况，如已进入签约或履约阶段。

如果同一关键词命中多个商机，Chatflow 不应直接分析，应返回对象选择卡，让用户选择本轮要分析的商机。

## 合同到现金链路

财务和合同相关 Agent 需要理解合同后的现金链路：

```text
合同 Contract
→ 回款计划 PaymentPlan
→ 回款记录 PaymentRecord
→ 发票 Invoice
```

常见判断：

- 已签合同是否有回款计划。
- 回款计划是否到期但未回款。
- 已回款是否已开票。
- 已开票是否仍未回款。
- 合同金额、计划金额、实际回款金额是否存在明显缺口。

## Chatflow 使用规则

- 普通问答不强制读取完整 L2C 链路。
- 客户/商机查询先判断是否唯一命中对象。
- 专项分析只读取本任务必要对象，不把全部 CRM 数据塞进 Prompt。
- 轻量商机评审可读取客户、商机、报价、付款、交付、合同摘要。
- 写回前必须确认唯一客户或商机。
- 文件上传只作为本轮补充材料，不改变客户或商机绑定关系。

## 输出要求

在 多Agent智能助手 工作台中，不直接展示底座对象关系图。应把读取行为转成用户能理解的判断过程，例如：

```text
判断过程
- 已识别为商机付款风险分析
- 已读取当前商机和所属客户资料
- 已检查报价、付款条件和最近跟进
- 已参考合同与回款链路规则
- 已生成下一步沟通建议
```
