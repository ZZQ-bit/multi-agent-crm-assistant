更新时间：2026-07-19

## 1. 执行结论

本轮按照“面试演示项目可用”标准完成了静态校验、后端测试、Tool API、40 条 Dify API 用例、12 条真实前端用例、安全边界和数据库前后置比对。

最终结论：**演示可用**。

| 指标 | 结果 | 验收要求 | 判断 |
| --- | ---: | ---: | --- |
| Dify API 有效返回 | 40/40，100% | 不低于 80% | 通过 |
| API 自动质量评分 | 平均 98.2/100 | 不低于 65 | 通过 |
| 前端核心场景 | 12/12，100% | 不低于 90% | 通过 |
| 前端失败恢复 | 1 条首次失败，点击重试后成功 | 可恢复 | 通过 |
| 后端单测 | 69/69 | 无阻断失败 | 通过 |
| Chatflow 静态校验 | 54 节点、22 个 Code 节点 | 结构断言通过 | 通过 |
| 本地/公网 Tool API | 两套冒烟均通过 | 可访问、鉴权有效 | 通过 |
| 数据库写回 | 前后状态签名一致 | 不允许意外写回 | 通过 |
| 内部协议泄露 | 未发现 | 不允许泄露 | 通过 |

自动评分是按已确认的宽松、信号型规则执行，主要用于统一检查任务完成、关键事实、业务建议、展示和只读边界，不等同于生产级模型评测。

### 3.1 首轮与回归

首轮 Dify API 执行结果为 35/40 有效返回。失败原因包括：

- Blocking 请求模式导致长流程 504，后改为 SSE streaming。
- 部分最终结果只存在于 `workflow_finished.outputs`，旧执行器只读取 `message.answer`。
- 长连接偶发 TLS/curl 连接中断。
- `PLAN-001` 首轮虽然有响应，但错误降级为信息不足。

修复测试执行器后，对 `SUM-001`、`PLAN-001`、`SPEC-001`、`SPEC-006`、`REV-003`、`REV-005` 六条用例低频回归，6/6 通过。原始首轮结果与回归结果均保留，最终报告使用合并后的结果。

### 3.2 质量观察

- 查询、总结、内容生成、专项和评审均能使用现有 CRM 对象，不再出现占位客户或商机。
- 多候选查询能返回南京数据治理中心的两个真实商机，不擅自选择。
- 高风险评审能识别 75 折、8 折、30% 首付、分期、价格保护、延期赔付和交付拆期等真实业务条件。
- 证据不足场景会列出缺口，没有补造付款、上线周期和合同条款。
- 草稿场景明确“未保存”或“不要写入”，数据库比对也证明没有写回。
- 外部情报能结合海尔智家、京东集团现有 CRM 商机，但公开资料内容具有动态性。

`CRM-005` 和 `CRM-006` 的回答只能提供统计汇总，不能列出满足金额或阶段条件的商机明细，自动评分均为 74 分。回答没有编造数据，但没有完全完成用户任务。

## 5. 安全与边界结果

- 错误 Tool Token 返回 `UNAUTHORIZED`。
- 无效商机 ID 返回 `CRM_READ_FAILED / 商机不存在`，没有返回假成功。
- 无效写回参数返回 `WRITEBACK_VALIDATION_FAILED`。
- 前端 12 条用例未出现 `processEvents`、`route_label`、`DIFY_TOOL_TOKEN` 等内部字段。
- 内部调试信息专项测试没有泄露路由、Agent 或节点输出，但回答只是泛化为“信息不足”，没有明确说明拒绝内部调试信息，判为需优化。

## 6. 缺陷与遗留项

| 编号 | 级别 | 问题 | 当前状态 | 演示影响与规避 |
| --- | --- | --- | --- | --- |
| DEF-01 | P1 | Dify Cloud 长流程存在超时、TLS 或长连接中断波动 | 回归通过，持续观察 | 演示前预热；避免并发；失败时点击重试 |
| DEF-02 | P2 | 金额/阶段全量筛选只有汇总，缺少商机明细 | 未修复 | 演示使用客户或商机精确查询，不使用全量筛选作为主链路 |
| DEF-03 | P2 | 复杂评审响应慢，桌面最长约 129 秒 | 未修复 | 提前准备问题；允许展示判断过程；保留备用短问题 |
| DEF-04 | P2 | 390px 移动端首屏被导航和历史栏占据 | 未修复 | 面试只使用桌面端演示 |
| DEF-05 | P2 | 内部调试信息请求未明确拒绝，只返回信息不足 | 未修复 | 未泄露信息，不影响主演示 |
| DEF-06 | P2 | Dify `/messages/{id}` traces 查询返回 404/SSL，无法稳定采集节点明细 | 未修复 | 使用静态拓扑、回答内容和后端日志作为证据 |
| DEF-07 | P2 | YAML 中存在本地演示 Tool Token 默认值 | 未修复 | 仅用于本地演示；公开部署前必须移出文件并轮换 |

本轮没有发现 P0 缺陷，没有发生真实误写、系统性 CRM 事实编造或密钥在用户回答中泄露。

## 7. 测试资产改进

本轮同时修复了测试基础设施：

- 40 条占位用例全部替换为真实 CRM 对象和 ID。
- Dify API 执行器改为 SSE streaming，支持长流程。
- 增加 `workflow_finished.outputs` 最终结果解析和 curl 连接中断兼容。
- 增加前后数据库快照、结果合并、宽松评分和失败用例定向回归脚本。
- 修正 Chatflow 静态校验中的两条过期 Prompt 文案断言。

所有原始结果位于 `output/acceptance/`，可复查本报告中的数字和结论。40 条业务用例的问题、用户可见回复全文、评分与耗时已整理至 `docs/17-多Agent智能助手 测试问题与完整回复记录.md`。
