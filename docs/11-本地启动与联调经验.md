# 本地启动与联调经验

更新时间：2026-07-06

## 推荐启动顺序

1. 启动 MySQL
2. 启动 Redis
3. 启动 `CordysCRM` 后端 Spring Boot
4. 确认本地后端可访问：`http://localhost:8081/is-login`
5. 启动 ngrok 固定域名 tunnel
6. 在 Dify Cloud 中确认 `CRM_TOOL_BASE_URL` 指向 ngrok 固定地址
7. 打开 `http://localhost:8081/#/login`
8. 进入 `http://localhost:8081/#/ai-deal-desk/index`

日常联调 `多Agent智能助手` 通常不需要单独启动前端 Vite。只要后端 `8081` 正常，CRM 页面可以直接从后端静态资源打开。

## 一键启动本地服务

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-local-deal-desk.ps1
```

这个脚本用于启动 MySQL、Redis 和后端。Dify Cloud 回调本机 Tool API 时，还需要单独保持公网 tunnel 运行。

## MySQL

- 可执行文件：`C:\Program Files\MySQL\MySQL Server 8.4\bin\mysqld.exe`
- 配置文件：`C:\opt\cordys\mysql\my.ini`

```powershell
& "C:\Program Files\MySQL\MySQL Server 8.4\bin\mysqld.exe" --defaults-file="C:\opt\cordys\mysql\my.ini" --console
```

## Redis

- 可执行文件：`<path-to-redis-server.exe>`
- 配置文件：`C:\opt\cordys\redis\redis.conf`
- 关键注意：这个 Windows 版 Redis 不能直接传 `C:\...`，需要传 `/cygdrive/c/...`

```powershell
& "<path-to-redis-server.exe>" "/cygdrive/c/opt/cordys/redis/redis.conf"
```

## 后端

工作目录：`<project-root>\CordysCRM`

```powershell
.\mvnw.cmd -pl backend/app spring-boot:run "-Dspring-boot.run.jvmArguments=-Dai.deal-desk.dify.tool-token=replace-with-your-own-token -Dai.deal-desk.dify.user-id=admin -Dai.deal-desk.dify.organization-id=100001"
```

说明：

- Phase 4 联调时，这组 JVM 参数默认不要省略。
- 它们用于让 Dify Cloud 通过匿名 gateway 调本地 Tool API 时，后端具备固定执行身份。

## 页面与登录

- 登录页：`http://localhost:8081/#/login`
- 多Agent智能助手 页面：`http://localhost:8081/#/ai-deal-desk/index`
- 默认账号：`admin`
- 默认密码：`CordysCRM`

## 本地自检

先看端口：

- `3306`
- `6379`
- `8081`

再看接口：

```powershell
Invoke-WebRequest -UseBasicParsing http://localhost:8081/is-login
```

如果返回 `200`，说明后端基本已经正常启动。

## Dify Cloud 联调

Dify Cloud 不能直接访问本机 `localhost`，必须通过公网 HTTPS tunnel 调本机 Tool API。

当前推荐使用 ngrok 固定域名，不再依赖每次都会变化的 `trycloudflare.com` quick tunnel。

当前固定 Tool API base URL：

```text
https://your-domain.example.com/anonymous/ai/deal-desk/dify-tools
```

Dify 环境变量应设置为：

```text
CRM_TOOL_BASE_URL=https://your-domain.example.com/anonymous/ai/deal-desk/dify-tools
DIFY_TOOL_TOKEN=replace-with-your-own-token
```

启动 ngrok：

```powershell
ngrok http --url=your-domain.example.com 8081
```

如果本机 ngrok 客户端提示 `--url` 不支持，可使用旧参数：

```powershell
ngrok http --domain=your-domain.example.com 8081
```

当前不再通过本地脚本回写 YAML 环境变量；如需调整，直接在 Dify Cloud 中更新应用环境变量。

如修改 Chatflow 文件，需要在 Dify Cloud 中重新导入 `chatflows/ai-deal-desk-v3.example.yml`，或手动更新已有应用的环境变量。

## 公网 Tool API 自检

本地后端启动、ngrok 运行后，可以用公网地址验证 Dify Tool API：

```powershell
$headers = @{
  "X-DIFY-TOOL-TOKEN" = "replace-with-your-own-token"
  "Content-Type" = "application/json"
}
$body = @{ keyword = "华东智造集团AI客服升级项目"; limit = 5 } | ConvertTo-Json -Compress
Invoke-WebRequest `
  -UseBasicParsing `
  -Method Post `
  -Uri "https://your-domain.example.com/anonymous/ai/deal-desk/dify-tools/search-opportunities" `
  -Headers $headers `
  -Body ([System.Text.Encoding]::UTF8.GetBytes($body))
```

返回 `200` 且包含 `success=true` / `code=OK`，说明 Dify Cloud 的 HTTP 节点也应能访问这个 Tool API。

## cloudflared 备用方案

`scripts\start-tunnel.ps1` 仍保留为备用方案，它会创建 Cloudflare quick tunnel：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-tunnel.ps1
```

注意：

- quick tunnel 地址通常是 `https://xxx.trycloudflare.com`
- 地址可能变化
- 可能被 Cloudflare 限流
- 出现 `429 Too Many Requests` / `error code: 1015` 时，不要继续排查 Chatflow，先恢复 tunnel

当前阶段优先使用 ngrok 固定域名，cloudflared 只作为临时备用。

## 常见排查顺序

1. `3306 / 6379 / 8081` 是否监听
2. MySQL / Redis 进程是否还活着
3. Spring Boot 日志里是否出现 `Tomcat started on port 8081`
4. `http://localhost:8081/is-login` 是否返回 `200`
5. ngrok 进程是否还在运行
6. Dify 环境变量 `CRM_TOOL_BASE_URL` 是否指向 `https://your-domain.example.com/anonymous/ai/deal-desk/dify-tools`
7. Dify HTTP 节点失败时，先用“公网 Tool API 自检”确认 tunnel 是否可用
8. 如果公网 Tool API 自检通过，再继续排查 Chatflow 节点、模型响应或后端 SSE 映射
